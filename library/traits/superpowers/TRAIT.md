# library/superpowers

本目录作为 `library/superpowers/*` 子 trait 的索引入口。

## 子 Trait 列表

- `superpowers/brainstorming`：在任何创造性工作前进行需求澄清与设计收敛。
- `superpowers/writing-plans`：面对多步骤任务时先产出可执行计划。
- `superpowers/executing-plans`：按既定计划执行并设置 review checkpoint。
- `superpowers/systematic-debugging`：遇到 bug/用例失败时先系统化定位再修。
- `superpowers/test-driven-development`：以测试驱动实现功能/修复。
- `superpowers/verification-before-completion`：宣称完成前先跑验证命令并用证据说话。
- `superpowers/dispatching-parallel-agents`：拆分并行子任务，提高吞吐。
- `superpowers/subagent-driven-development`：在当前 session 中用子任务分治执行。
- `superpowers/requesting-code-review`：提交/合入前请求代码审查。
- `superpowers/receiving-code-review`：接收评审反馈时先核实再改。
- `superpowers/finishing-a-development-branch`：收尾分支（PR/合并/清理）的结构化指引。
- `superpowers/using-git-worktrees`：用 git worktree 隔离开发环境。
- `superpowers/using-superpowers`：在对话开始时建立 skills 使用规则。
- `superpowers/writing-skills`：创建/编辑/验证 skills。
