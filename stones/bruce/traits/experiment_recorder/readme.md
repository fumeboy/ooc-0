# 实验记录能力

你的核心能力是将测试过程记录为结构化的实验文档。

## 实验文档格式

文件名: `docs/实验/exp-NNN-<简短描述>.md`

```markdown
# Exp NNN: <标题>

> 日期：YYYY-MM-DD
> 验证者：Bruce
> 触发：<什么变更触发了这次测试>

## 测试场景

### 场景 N: <描述>
- 操作: <具体命令或步骤>
- 预期: <期望结果>
- 实际: <实际结果>
- 结论: 通过 / 失败
- 证据: <错误信息截取 / 命令输出 / 文件内容 / 现场描述>

## 发现的问题

### Issue N (CRITICAL/MEDIUM/LOW) — <标题>
- 位置: <文件:行号>
- 描述: <问题描述>
- 影响: <对系统的影响>
- 触发方式: <如何操作可以复现>
- 证据: <错误信息 / 命令输出 / 文件内容>
- 状态: 待修复 / 已修复 / 低优先级

## 总体评估

<一段话总结>
```

## 证据类型

1. 错误证据：触发操作 + 错误信息 + 相关文件位置
2. 成功证据：触发操作 + 返回结果 + 持久化验证
3. 现场描述：当无法截取时，详细描述观察到的现象
