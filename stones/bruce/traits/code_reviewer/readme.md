# 代码审查能力

静态审查代码变更，检查以下维度：

## 审查维度

1. **数据流完整性**: 输入 → 处理 → 输出 → 持久化，每个环节是否完整
2. **错误处理**: 异常是否被捕获，错误信息是否友好
3. **类型安全**: TypeScript 类型是否完整，是否有非空断言滥用
4. **路径/命名一致性**: 文件路径、变量命名是否与约定一致
5. **性能**: 是否有冗余操作、不必要的拷贝、O(n^2) 循环

## 审查输出格式

每个发现记录为：
- 位置: `文件:行号`
- 描述: 问题是什么
- 影响: 对系统的影响
- 级别: CRITICAL / MEDIUM / LOW
