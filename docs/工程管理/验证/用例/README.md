## 用例目录结构

每个用例一个子目录：

```
用例001_向supervisor发消息/
├── README.md        ← 用例描述：步骤、期望结果
├── steps.md         ← 详细步骤
└── screenshots/     ← 截图（可选）
```

## 运行方式

用例**不是自动化脚本**——是手动执行的 checklist。

典型流程：

```
1. 启动后端：cd user && bun kernel/src/cli.ts start 8080
2. 打开前端：浏览器访问 localhost:8080
3. 按用例的步骤操作
4. 对照期望结果检查
5. 如有异常记录下来
```

## 新增用例

何时写新用例：

- 新功能上线 → 补对应用例
- 发现某个 regression → 为该场景写用例
- 哲学上验证某个涌现能力

## 用例 vs 单元测试

| 维度 | 用例 | 单元测试 |
|---|---|---|
| 粒度 | 端到端场景 | 单个函数 |
| 执行 | 手动（人或 Bruce） | 自动（bun test） |
| 速度 | 分钟级 | 毫秒级 |
| 关注 | 用户感受 | 代码正确 |

**互补，不可替代**。

## 未来：自动化

目前用例是手动执行的。未来可以用 Playwright 等工具做自动化——但要注意：

- 自动化脚本维护成本高
- 体验类问题（动画、感受）难以自动化

保守策略：**关键路径自动化（如 001、002），深度场景保留手动**。
