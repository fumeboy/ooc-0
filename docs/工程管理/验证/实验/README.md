# 实验 — 探索性验证记录

> 当需要验证一个**未确定的问题**时，做一次实验，把结果记录下来。

## 与用例的区别

| 维度 | 用例 | 实验 |
|---|---|---|
| 目的 | 验证已知功能 | 回答未知问题 |
| 次数 | 反复运行 | 通常一次性 |
| 产物 | Checklist | 实验报告（一次性）|

## 实验文件结构

```
exp-{NNN}-{主题短名}.md
```

典型内容：

```markdown
# Exp NNN: 主题

## 假设
[实验前你认为结果会是什么]

## 方法
[怎么做的实验]

## 观察
[实际看到了什么]

## 结论
[假设成立吗？推论是什么？]

## 对哲学的影响
[是否需要更新基因 / 涌现描述？]
```

## 实验与哲学的联动

实验的关键价值是**验证哲学主张**：

- **验证涌现**：某条涌现（`哲学/emergences/`）是否真的出现
- **验证基因**：基因的某个推论是否成立
- **发现反例**：出现了哲学没预测的行为？

反例是最有价值的——它们触发：

```
实验观察到反例
  ↓
写入 discussions/ 开始讨论
  ↓
形成共识 → 修改基因 或 修改实现
```

## 如何开始一个实验

1. **形成假设**：明确"我想验证 X"
2. **设计方法**：什么配置、什么输入、怎么观察
3. **执行**：实际运行，记录观察
4. **反思**：结论、局限、后续问题

写在一份 `exp-{NNN}-{name}.md` 文件里即可。
