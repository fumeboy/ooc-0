## 背景 / 问题描述

上一轮迭代（`20260421_feature_统一title参数清理child_title.md` Phase C）发现两个系统性问题：

1. **测试 CWD 假设危险**：`trait-git-ops.test.ts` 等多个测试隐性假设"从 user 仓根运行"，用 `process.cwd() + "kernel"` 构造路径。但 `kernel/` 本身是独立 git submodule——从 `kernel/` 下直接 `bun test` 会变成 `kernel/kernel` 导致失败。已修一个，但类似假设可能还存在于其他 trait 测试中。
2. **MockLLMClient.responses 半失效**：`kernel/src/thinkable/client.ts` 里的 `MockLLMClient` 原设计是给旧 TOML parser 喂字符串用的。TOML 路径删除后，`responses` 只能作 content 透传，不再被解析为指令。上一轮已扩展 `responseFn` 支持 `{content, toolCalls, thinkingContent}`，但 `responses: string[]` 字段仍挂在接口上，容易被新写测试误用。

（原分类还有一项"旧 Flow pause/resume"，已随"旧 Flow 架构退役"迭代处理，不在本迭代范围。）

## 目标

1. **CWD 独立性**：
   - 扫描 `kernel/tests/` 所有测试文件，找出依赖 `process.cwd()` 或硬编码相对路径的断言。
   - 统一改用 `import.meta.dir` + `path.resolve` 构造绝对路径。
   - 确保任意工作目录下 `bun test` 都通过。
2. **MockLLMClient 精简**：
   - 删除 `MockLLMClient` 的 `responses: string[]` 字段（或标记 deprecated 并加 warning）。
   - 检查 `kernel/tests/` 里还有哪些测试在用 `responses`，迁移到 `responseFn + toolCalls` 模式。
   - 更新 `MockLLMClient` 的注释和（如有）参考文档。

## 方案

### 阶段 A — CWD 独立性扫描

1. **扫描**：
   ```
   rg -n 'process\.cwd\(\)' kernel/tests/ kernel/src/
   rg -n 'from\s+["\'].\./.\./' kernel/tests/
   ```
   列出每个匹配的文件和行号，判断是否是 CWD 依赖。

2. **分类**：
   - **测试 bug**（测试本身依赖 cwd）：改用 `import.meta.dir` 拼绝对路径。
   - **实现 bug**（src 代码依赖 cwd）：评估是否需要接收显式 rootDir 参数。如果 src 代码是故意读 `process.cwd()`（如 cli.ts 确定 OOC_ROOT），保留。

3. **修复验证**：
   - 从 `kernel/` 目录跑 `bun test` → 0 fail
   - 从 `user/` 目录跑 `cd kernel && bun test` → 0 fail
   - 从 `/tmp` 下跑 `cd /Users/zhangzhefu/x/ooc/kernel && bun test` → 0 fail
   - 三种工作目录结果一致。

### 阶段 B — MockLLMClient 精简

1. **读代码**：`kernel/src/thinkable/client.ts` 的 `MockLLMClient` 实现。确定 `responses: string[]` 当前语义和被调用点。

2. **扫描使用**：
   ```
   rg -n 'MockLLMClient.*responses:' kernel/tests/
   rg -n 'new MockLLMClient' kernel/tests/
   ```
   列出每个使用 `responses` 字段的测试。

3. **迁移每个**：
   - 如果测试只是字符串透传（不指望被 parser 处理）→ 改用 `responseFn` 返回 `{ content: "..." }`。
   - 如果测试期待 tool call → 改用 `responseFn + toolCalls`（参考 `tests/thread-title.test.ts`）。

4. **删除字段**：迁移完成后，从 `MockLLMClient` 构造参数/字段中删除 `responses: string[]`，更新 JSDoc。

5. **验证**：`bun test` 0 fail。
