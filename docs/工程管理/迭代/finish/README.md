# finish/ — 已完成的迭代项

> 本目录**只存软链接**，指向 `../all/` 中的文件。

## 语义

- 出现在这里 = 已完成（含验证）
- 不会再变动（除非 re-open，回到 `doing/`）

## 完成的标准

一个 feature / bugfix 进入 finish 的前提：

1. **代码已实现**
2. **文档已同步**（相关架构文档、meta.md 交叉引用）
3. **测试通过**（单元测试 + 涉及的用例）
4. **验证完成**（at least 一次人工或 Bruce 体验）
5. **all/ 下的文档补全执行记录**（关键决策、遇到的问题、最终结果）

## 浏览已完成

```bash
ls finish/ | sort -r    # 按日期倒序，最近完成在前
```

## 归档（可选）

如果 finish/ 文件过多（上百个），可按季度归档：

```
finish/
├── 2026Q1/
├── 2026Q2/
└── 2026Q3/
```

但 `all/` 保持扁平不归档——它是物理真相源。
