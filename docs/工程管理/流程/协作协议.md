# 协作协议 — 多部门协作时的信息传递规则

> 当任务涉及多个部门时，如何保证信息准确、快速地传递？

## 三条基本协议

### 1. 单一真相源（Single Source of Truth）

每份信息**只有一个权威来源**：

| 信息类型 | 权威来源 |
|---|---|
| 总体目标 | Session 的 Issue description |
| 本部门任务 | 对应 Task description |
| 讨论记录 | Issue 的 comments |
| 设计方案 | 设计文档（Sophia 产出） |
| 代码现状 | 代码本身 + `对象/` 下的架构文档 |
| 进度 | Task status + subtasks |

**不要重复存储同一信息**。如果两个地方的信息冲突，以权威来源为准。

### 2. 变更必须通知

某个部门的工作**依赖于**另一个部门的产出，但产出发生变更：

- 产出部门 **主动通知** 依赖部门（通过 talk）
- 附上变更内容 + 影响说明
- 依赖部门评估是否需要调整

例：
```
Sophia: [完成设计方案 v2，与 v1 差异：...]
  → talk(kernel-agent, "设计方案有变，请看新的文档")
  → talk(iris-agent, "设计方案有变，评估对前端的影响")
```

### 3. 阻塞不沉默

某部门**卡住**（等待外部决策 / 发现需求冲突 / 其他资源问题）：

- **立即** 在相关 Issue 下评论，说明阻塞原因
- **@supervisor** 请求协调
- 不要默默等待

## 协作模式

### 模式 A：串行交接

```
Sophia(设计) → Kernel(实现) → Iris(前端)
```

交接点：
- Sophia 完成后，写一份**接口文档**（API 形态、数据结构）
- Kernel 基于接口文档开始
- Kernel 完成后，更新接口文档（实际实现可能与设计略有偏差）
- Iris 基于更新后的接口文档开始

### 模式 B：并行协作

```
Sophia(持续哲学)
Kernel(后端实现)
Iris(前端实现)  ← 都同时推进
```

需要：
- 每周一次**同步会**（可异步：在 Issue 下写进度）
- 重大决策走 Issue 讨论（不私下决定）
- 接口提前约定（避免后期大改）

### 模式 C：迭代闭环

```
Sophia → Kernel → Iris → Bruce(测试) → Sophia(反思) → ...
```

小步快跑，每轮产出可体验的功能。用于探索性任务。

## 协作中的常见问题

### 问题 1：部门对需求理解不一致

**症状**：Alan（Kernel）和 Iris 都在做 X，但做的是"相似但不同"的东西。

**原因**：Supervisor 分发时没有把任务描述清楚。

**解法**：
- Supervisor 复盘 Task 描述
- 在 Issue 下说明"我理解的 X 是什么"
- 两个部门在 Issue 下评论对齐

### 问题 2：依赖迟迟不来

**症状**：Iris 在等 Kernel 的 API，但 Kernel 延期了。

**解法**：
- Iris 在 Issue 评论："我的 P0 在等 Kernel 的 X API"
- Supervisor 评估：是否该调整优先级？或 Iris 先做其他 P1？

### 问题 3：某部门产出质量不达标

**症状**：Kernel 交付的 API 有明显 bug，但 Iris 已开始集成。

**解法**：
- Iris 暂停集成，在 Issue 下反馈具体问题
- Supervisor 决定：是 Kernel 修复还是 Iris 先 workaround
- 不要私下 patch（容易掩盖问题）

## 信息密度指南

### Issue comment：中等密度

- 每次有进展时评论（每天 1-3 条）
- 包含：做了什么、结论是什么、下一步
- **不包含**：细碎的 thought process（那是 Session 内部的事）

### 设计文档：高密度

- 全面讲清 "why / what / how"
- 图示、表格、代码示例
- 放在 `对象/` 或 `哲学/discussions/` 下

### talk 消息：低密度

- 快速沟通、问答
- 不替代 Issue 评论（因为不可追溯）
- 重要信息最终要沉淀到 Issue

## 与 OOC 本身的对称

OOC 的对象协作（[../../对象/合作/](../../对象/合作/)）和我们（人+AI）的协作本质相同：

| OOC 对象协作 | 我们的协作 |
|---|---|
| Session | 一次任务周期 |
| Issue | Issue（就是同一个东西） |
| talk | Slack / 邮件 / 口头 |
| 看板 | 看板（同一个东西） |
| Supervisor | Supervisor（同一个对象） |

**我们用 OOC 的方式做 OOC**——协作协议在人和对象之间是通用的。
