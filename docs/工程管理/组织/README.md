> OOC 的前后端工程基本完善，但根本难题尚未攻克。
> 单人全栈模式已触及天花板——不是能力不够，而是**注意力带宽不够**。
> 一个人同时思考哲学、写代码、调 UI、搭生态，每个方向都只能浅尝辄止。
>
> 解法：**分层聚焦，各司其职。**

---

## 模型概览

```
                    ┌─────────────────────┐
                    │   最高哲学设计层 (1)  │
                    │      Sophia          │
                    │   "为什么" 的守护者    │
                    └──────────┬──────────┘
                               │ 哲学指导 ↓ ↑ 实践反馈
              ┌────────────────┼────────────────┐
              │                │                │
    ┌─────────▼──────┐ ┌──────▼────────┐ ┌─────▼──────────┐
    │ OOC 核心思想层  │ │ OOC 用户体验层 │ │ OOC 生态搭建层  │
    │    Kernel       │ │    Iris        │ │    Nexus        │
    │  思考循环+社交网 │ │  人类视角+UI   │ │  扩展能力+连接  │
    └────────────────┘ └───────────────┘ └────────────────┘
```

**1 = 最高哲学设计层（Sophia）**
深入思考 OOC 的哲学设计，维护 gene.md / emergence.md / discussions.md，为三个执行层提供方向指引和设计决策。

**3 = 三个执行层**

| 层 | 代号 | 核心职责 |
|----|------|---------|
| 核心思想层 | Kernel | 思考循环、对象社交网络、认知栈工程实现、测试、问题报告 |
| 用户体验层 | Iris | 从人类角度审视 OOC 的 UI/UX，让系统"好用、好看、好懂" |
| 生态搭建层 | Nexus | 为 OOC 增加扩展能力（邮箱、浏览器、终端、IM 等），生产功能对象 |

---

## 协作关系

### 纵向：哲学层 ↔ 执行层

```
Sophia ──哲学指导──→ Kernel / Iris / Nexus
Sophia ←──实践反馈── Kernel / Iris / Nexus
```

- Sophia 不写代码，只输出哲学文档和设计决策
- 三个执行层遇到根本性问题时，向 Sophia 提交「哲学咨询」
- Sophia 的回答更新到 gene.md / emergence.md / discussions.md
- 执行层根据更新后的哲学文档调整工程方向

### 横向：执行层之间

```
Kernel ←→ Iris    : Kernel 提供 API 和数据模型，Iris 提供 UI 需求和体验反馈
Kernel ←→ Nexus   : Kernel 提供 Trait/Effect 机制，Nexus 基于此构建扩展
Iris   ←→ Nexus   : Nexus 的扩展需要 UI 入口，Iris 设计交互方式
```

每个执行层都运行自己的内循环，同时参与全局的外循环：

```
┌──────────── 全局外循环（Sophia 驱动）────────────┐
│                                                  │
│  哲学思考 → 更新文档 → 指导执行层                   │
│       ↑                    ↓                     │
│  汇总反馈 ←── 三层各自完成一轮内循环               │
│                                                  │

│  │ 调研→设计→实现→测试→反馈 │                      │

│  │ 需求→设计→实现→集成→验证 │                      │

└──────────────────────────────────────────────────┘
```

## 为什么是 1+3 而不是 4

哲学层不是"第四个执行层"。它是**元层**——它思考的是"OOC 应该是什么"，而不是"OOC 怎么做"。

三个执行层回答"怎么做"：
- Kernel：怎么让对象更聪明
- Iris：怎么让人类更舒服
- Nexus：怎么让系统更有用

Sophia 回答"为什么"：
- 为什么对象需要认知栈？
- 为什么遗忘是智能的基础设施？
- 为什么 Bias > Role？

**没有"为什么"，"怎么做"就是盲目的。没有"怎么做"，"为什么"就是空谈。**
1+3 的结构确保两者始终同步。
