# 理想：OOC 作为终极智能 Agent 系统

> 计算机革命还没有发生。我们仍在用 1960 年代的思维模式操作计算机。
> 真正的革命是：计算机不再是工具，而是一个有生命的生态系统。
> —— Alan Kay

---

## 什么是「终极智能 Agent」

传统 Agent 系统的天花板是 LLM 的能力上限。
一个 Agent 再怎么精心编排 prompt，它的智能不会超过底层模型。

OOC 的野心更大：**让系统整体的智能涌现出超过任何单个模型的能力。**

这不是幻想——人类社会就是这样工作的：
- 单个人的智力有限
- 但人类文明的集体智慧远超任何个人
- 关键在于：**结构化的协作 + 知识的积累 + 角色的分工**

OOC 要做的就是把这个模式复制到 AI 系统中。

OOC 的哲学定义写在 `docs/哲学/genes/` 中。
并且 `docs/哲学/emergences/` 推演了 OOC 系统的能力表现。

> **核心哲学 G12（经验沉淀原则）**：LLM 总是解决思考问题，但它可以沉淀自己的经验、套路、常用模式、工具为文档、方法程序，来达到「不为重复的问题浪费思考」的理想状态。
> 详见 [g12-经验沉淀.md](../../哲学/genes/g12-经验沉淀.md)。
>
> **核心哲学 G13（认知栈）**：对象的运行时本质是一个混合栈——每个栈帧同时包含过程和思维。
> Trait 自动激活、结构化遗忘、经验沉淀都是栈帧 push/pop 的自然副作用。智慧 = 帧 0 的厚度。
> 详见 [g13-认知栈即运行模型.md](../../哲学/genes/g13-认知栈即运行模型.md)。

## 理想 1: Deep Research —— 像研究员一样工作

我希望 OOC 能完成这样的任务：

> 「研究 Alan Kay 对面向对象编程的原始设想，与当代 OOP 实践的偏离，
>   以及这种偏离对软件工程造成的影响。输出一份万字报告。」

一个理想的 OOC Deep Research 流程：

```
User 发起请求
  → Supervisor 分析任务，创建 Research Flow
    → Research Flow 制定研究计划（plan.md）
      → 创建子 Thread: "信息收集"
        → 使用 browser 扩展搜索论文、博客、采访
        → 将关键信息摘录到 windows/notes.md
      → 创建子 Thread: "文献分析"
        → 激活 critical_thinking_bias
        → 对收集的信息进行批判性分析
        → 识别矛盾点和关键争议
      → 创建子 Thread: "报告撰写"
        → 激活 writing_bias（尚不存在）
        → 综合分析结果，撰写结构化报告
        → 创建 report 对象，写入报告内容
    → Research Flow 汇总子 Thread 结果
    → 更新 plan.md 标记完成
    → sendMessage 给 User，附上报告链接
```


## 理想 2: 自主进化 —— 经验沉淀

> **注意**：自我进化的核心不是「修复错误」，而是「沉淀经验」。
> 基础错误已被 OUTPUT_BIAS 覆盖，真正的价值在于从成功/失败中提炼可复用的模式。

理想状态：

```
一个 Flow 执行新类型的任务
  → 第一次：深度思考，尝试多种方法，最终成功
  → 反思：这个任务的模式是什么？哪些方法是有效的？
  → 沉淀：使用 writeBias() 创建新的 bias 来记录这个模式
  → 第二次遇到类似任务
  → 新 bias 被激活，直接应用沉淀的模式
  → 不再需要从头思考
  → 思考能力被释放给更新的问题
```

另一个场景（从失败中学习）：

```
一个 Flow 执行任务 → 失败了 3 次
  → 自动激活 debugging_bias（已实现）
  → 反思失败原因
  → 发现是缺少某种能力
  → 自己编写新的代码方法
  → 自己更新自己的 bias（从失败中提炼教训）
  → 用新能力重新执行任务
  → 成功
  → 这些改进被持久化，下次不会再犯
```


## 理想 3: 多对象协作 —— 涌现集体智慧

理想状态：

```
World 中有多个专业对象：
  - researcher: 善于信息收集和分析
  - writer: 善于组织语言和撰写文档
  - critic: 善于批判性审查
  - coder: 善于编写和测试代码

当 User 说「写一篇关于 AI 安全的深度文章」：
  → Supervisor 分配给 researcher
  → researcher 搜索信息，整理要点，talk 给 writer
  → writer 撰写初稿，talk 给 critic
  → critic 审查，指出问题，talk 回 writer
  → writer 修改，再次 talk 给 critic
  → 直到 critic 满意
  → 最终成品 talk 给 User
```


## 理想 4: 人机深度协作

不是「人类命令 AI 执行」，而是「人类和 AI 对象共同思考」：

```
Human 暂停一个 Flow
  → 查看它的 Thread 树，理解它的思考过程
  → 注入一个 thought: "你忽略了 X 因素，请重新考虑"
  → 恢复 Flow
  → Flow 看到注入的 thought，调整策略
  → 继续执行，产出更好的结果
```

## 理想 5: 系统自我构建 —— OOC 开发 OOC

> 终极目标不是人类不断优化 OOC，而是 OOC 系统**完全替代人类**完成自身的构建与优化。

**这不是"对象优化自己的 bias"——那是 G12 已经做的事。**
这是：OOC 系统像一个开发者一样，**修改 `./src` 源代码、编写 `./docs` 文档、运行测试、设计架构、做实验**。


## 方法论：实验驱动

Alan Kay 说过：「观点值 80 IQ。」

同样的道理：**不是先设计完美架构，再去实现；而是先实验，再从实验中发现真正的问题。**

实验记录由 Bruce（OOC 系统的真实用户）维护，写入 `docs/实验/` 目录。
优先级和进度跟踪见 [路线图/README.md](README.md)。
