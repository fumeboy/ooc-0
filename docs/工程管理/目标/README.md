# 目标 — 我们要做什么

> 工程管理元循环的**起点**。任何工作开始前都要回答：现在做什么，为了什么。

## 三个文档

| 文档 | 粒度 | 更新频率 |
|---|---|---|
| [愿景.md](愿景.md) | 终极目标（"打造世界上最聪明的 Agent 系统"） | 半年级 |
| [现状.md](现状.md) | 当前离理想多远 | 月级 |
| [当前迭代.md](当前迭代.md) | P0 / P1 / P2 优先级 | 周级 |

## 三层粒度的用处

### 愿景（半年级）

提供**方向**。回答："我们总体想达到什么"。

愿景稳定，不常变。当有重大方向讨论时才更新。

### 现状（月级）

提供**距离感**。回答："我们现在离愿景多远？哪些路径走通了？哪些还未？"

一个月评估一次：哪些领域有突破，哪些仍是瓶颈。

### 当前迭代（周级）

提供**执行清单**。回答："这周、这个迭代，具体做什么？"

按 P0 / P1 / P2 优先级排列。完成一个 move 下一个。

## 与"对象/"的呼应

目标不是"写在墙上"就够了。它需要**进入对象的 Context**：

- **Supervisor** 的 readme 应该引用当前愿景
- **Supervisor** 的 data.json 应该跟踪当前迭代进展
- 每次新 Session 开始，Supervisor 能读到最新的优先级

这让"目标"不是抽象概念，而是**具体落入每次工作的 Context**。

## 目标的演化

目标会变——通过**反思** 修正：

```
当前迭代结束：
  → 评估完成度
  → 评估是否该推进到下一个 P0
  → 更新现状
  → 如需要，调整愿景
```

这个过程对应 OOC 的沉淀循环——反思让目标进化。
