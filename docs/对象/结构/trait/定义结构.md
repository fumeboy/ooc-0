# Trait 的定义结构

> TraitDefinition 是 trait 的内部表示。一个 trait 文件（TRAIT.md + methods.ts）加载后会变成一个 TraitDefinition 对象。

## TraitDefinition 字段

```typescript
interface TraitDefinition {
  // 基本信息
  name: string;              // 完整路径名，如 "kernel/computable/file_ops"
  description: string;        // 能力描述（注入 Context 让 LLM 理解）
  readme: string;             // TRAIT.md 内容（激活时注入 Context）

  // 指令关联
  command_binding: string[];  // 关联的指令列表（open 时自动加载）

  // 方法
  methods: MethodDef[];       // 注册方法（沙箱中可调用的函数）

  // 树形结构
  children: string[];         // 子 trait ID 列表（树形时自动填充）
  parent: string | null;      // 父 trait ID（树形时自动填充）

  // 生命周期
  when: "always" | "never" | "custom";  // 激活策略
}
```

## 字段详解

### name

完整路径名。用斜杠分隔层级：

```
kernel/base                             ← 顶级 trait
kernel/computable                        ← 顶级 trait
kernel/computable/file_ops               ← 子 trait
kernel/computable/file_ops/glob          ← 孙 trait（如有）
```

命名空间规则：
- `kernel/` — 内置 Kernel Traits（位于 `kernel/traits/`）
- `library/` — 公共 Library Traits（位于 `library/traits/`）
- `stone/{name}/` — 对象自定义 Traits（位于 `stones/{name}/traits/`）

### description

**一行描述**。当父 trait 被激活时，其子 trait 的 description 对 LLM 可见（让它知道"有哪些进一步能力可以激活"）。

例：`kernel/computable/file_ops` 的描述可能是 `"文件读写、目录列表、原子写入等文件操作能力"`。

### readme

TRAIT.md 文件的**完整内容**。当 trait 被激活（Level 3）时，这段内容注入 Context 的 `knowledge` 区。

典型 TRAIT.md 结构：

```markdown
---
name: computable
description: 代码执行能力（program 指令）
command_binding: [program]
when: never
---

# Computable — 代码执行能力

你可以通过 `program` 指令执行代码。沙箱环境提供以下 API：
- `readFile(path)` — 读文件
- `writeFile(path, content)` — 写文件
- ...

## 核心原则
...
```

### command_binding

当 `open(type=command, command=X)` 时，**自动激活**该 trait。

例：`kernel/computable` 的 `command_binding: [program]` 意味着任何对象 `open(command=program)` 时，`kernel/computable` 会自动加载。

一个 command 可以触发多个 trait 激活（如果多个 trait 都 binding 到同一 command）。

### methods

注册的沙箱方法列表。每个方法：

```typescript
interface MethodDef {
  name: string;
  description: string;
  params: ParamDef[];
  handler: (ctx, args) => Promise<any>;
}
```

只有当 trait 处于激活状态时，其 methods 才在沙箱中可调用。

详见 [方法注册.md](方法注册.md)。

### children / parent

**运行时自动填充**。加载 trait 时，如果发现父 trait，则双向建立关系。

例：加载 `kernel/computable/file_ops` 时：
- `file_ops.parent = "kernel/computable"`
- `kernel/computable.children` 追加 `"kernel/computable/file_ops"`

### when

激活策略：

| 值 | 含义 |
|---|---|
| `always` | 对象 readme 激活此 trait → 一直注入，永不卸载 |
| `never` | 不会主动激活，只能通过 command_binding 或 open(type=trait) 加载 |
| `custom` | 自定义条件（很少用） |

**基座层**（如 `kernel/base`）通常是 `always`；**能力层**（如 `kernel/computable`）通常是 `never`。

## 文件结构

一个 trait 在文件系统中的表示：

```
kernel/traits/computable/              ← 目录名 = trait 名
├── TRAIT.md                           ← 主 readme + frontmatter
├── methods.ts                         ← 方法实现（可选）
├── file_ops/                          ← 子 trait
│   ├── TRAIT.md
│   └── methods.ts
├── file_search/
│   ├── TRAIT.md
│   └── methods.ts
└── ...
```

加载过程：遍历 `kernel/traits/`，每个含 TRAIT.md 的目录 = 一个 trait。子目录递归处理，自动建立父子关系。
