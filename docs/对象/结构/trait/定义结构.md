# Trait 的定义结构

> TraitDefinition 是 trait 的内部表示。一个 trait 目录（TRAIT.md / VIEW.md + index.ts / backend.ts）加载后会变成一个 TraitDefinition 对象。
> 2026-04-27：knowledge frontmatter 删除 `when`，`activates_on` 分成描述展示与正文展示两类。

## TraitDefinition 字段

```typescript
type TraitNamespace = "kernel" | "library" | "self";
type TraitKind = "trait" | "view";

interface TraitDefinition {
  // 身份（2026-04-21 改造）
  namespace: TraitNamespace;       // 必填，frontmatter 显式声明
  name: string;                    // namespace 下的相对名（如 "computable", "computable/file_ops"）
  kind: TraitKind;                 // "trait" | "view"（默认 trait；VIEW.md 自动 view）

  // traitId = `${namespace}:${name}`（由 activator.traitId(trait) 组装）

  // 基本信息
  description: string;             // 能力描述（注入 Context 让 LLM 理解）
  readme: string;                  // TRAIT.md / VIEW.md 内容（激活时注入 Context）
  type: "how_to_think" | "how_to_use_tool" | "how_to_interact";

  // 指令关联：分别声明只展示描述、或展示正文内容的触发路径
  activatesOn?: {
    showDescriptionWhen?: string[];
    showContentWhen?: string[];
  };

  // 方法（双通道）
  llmMethods?: Record<string, TraitMethod>;   // LLM 沙箱可见（callMethod 入口）
  uiMethods?: Record<string, TraitMethod>;    // HTTP 可见（POST /call_method 入口）

  // 依赖
  deps: string[];                  // 依赖 traitId 列表（可省略 namespace：按 self→kernel→library 解析）

  // 树形结构（运行时自动填充）
  children?: string[];
  parent?: string;

  // Hooks
  hooks?: Partial<Record<TraitHookEvent, TraitHook>>;

  // 目录路径
  dir: string;                     // 物理目录（用于 readTrait 等运行时查询）
}
```

## 字段详解

### namespace + name → traitId

traitId = `${namespace}:${name}`（冒号分隔）。三个合法 namespace：

| namespace | 物理来源 | 示例 traitId |
|---|---|---|
| kernel | `kernel/traits/**` | `kernel:computable`、`kernel:computable/file_ops` |
| library | `library/traits/**` | `library:lark/doc` |
| self | `stones/{name}/traits/**` + `stones/{name}/views/**` | `self:reporter`、`self:main`（后者为 view）|

name 在 namespace 下可含 `/` 分级（形成树形结构）。
**frontmatter 必须显式声明 namespace + name**，loader 不再从路径推断。

### kind

- `"trait"` — 普通 trait（默认）
- `"view"` — UI 视图（kind=view 的 trait），对应 views/{viewName}/VIEW.md

VIEW.md 文件名自动触发 kind="view"（即使 frontmatter 未显式声明）。

### description

**一行描述**。当父 trait 被激活时，其子 trait 的 description 对 LLM 可见（让它知道"有哪些进一步能力可以激活"）。

例：`kernel/computable/file_ops` 的描述可能是 `"文件读写、目录列表、原子写入等文件操作能力"`。

### readme

TRAIT.md 文件的**完整内容**。当 trait 被激活（Level 3）时，这段内容注入 Context 的 `knowledge` 区。

典型 TRAIT.md 结构：

```markdown
---
name: computable
description: 代码执行能力（program 指令）
activates_on:
  show_content_when: [program]
---

# Computable — 代码执行能力

你可以通过 `program` 指令执行代码。沙箱环境提供以下 API：
- `readFile(path)` — 读文件
- `writeFile(path, content)` — 写文件
- ...

## 核心原则
...
```

### activates_on.show_description_when / show_content_when

当 `open(title="...", type=command, command=X, description="...")` 时，Knowledge Activator 会根据当前 command path 决定展示形态。

- `show_description_when`：只展示 `description` / summary，让 LLM 知道"有这个知识"，但不把正文注入 Context。
- `show_content_when`：展示 `TRAIT.md` / `VIEW.md` 正文内容，并进入完整能力激活路径。

例：`kernel/computable` 的 `show_content_when: [program]` 意味着任何对象 `open(title="执行程序", command=program, description="...")` 时，`kernel/computable` 会自动加载完整正文。

一个 command path 可以触发多个 knowledge；同一个 knowledge 也可以在不同 path 下选择不同展示形态。

### llmMethods / uiMethods

双通道方法表（2026-04-21）。每个方法：

```typescript
interface TraitMethod {
  name: string;
  description: string;
  params: TraitMethodParam[];
  fn: (ctx: MethodContext, args: object) => Promise<any>;  // 第二个参数永远是对象
  needsCtx?: boolean;   // 默认 true
}
```

- `llmMethods` 由 `index.ts` / `backend.ts` 的 `export const llm_methods = { ... }` 装入
- `uiMethods` 由同一文件的 `export const ui_methods = { ... }` 装入
- LLM 沙箱通过 `callMethod(traitId, method, args)` 调 `llmMethods`
- 前端通过 HTTP `POST /api/flows/:sid/objects/:name/call_method` 调 `uiMethods`（白名单严格）

详见 [方法注册.md](方法注册.md)。

### children / parent

**运行时自动填充**。加载 trait 时，如果发现父 trait，则双向建立关系。

例：加载 `kernel/computable/file_ops` 时：
- `file_ops.parent = "kernel/computable"`
- `kernel/computable.children` 追加 `"kernel/computable/file_ops"`

## 文件结构

### 普通 Trait

```
kernel/traits/computable/              ← 目录名（与 frontmatter.name 对应）
├── TRAIT.md                           ← 主 readme + frontmatter（必填 namespace + name）
├── index.ts                           ← 方法实现（可选）：export const llm_methods / ui_methods
├── file_ops/                          ← 子 trait
│   ├── TRAIT.md
│   └── index.ts
├── file_search/
│   ├── TRAIT.md
│   └── index.ts
└── ...
```

### View（kind=view）

```
stones/{name}/views/{viewName}/
├── VIEW.md                            ← frontmatter: namespace=self, kind=view
├── frontend.tsx                       ← 必须存在（loader 强制校验）
└── backend.ts                         ← 可选；与普通 trait 的 index.ts 同契约
```

Flow 级 views：`flows/{sid}/objects/{name}/views/{viewName}/`（同结构，覆盖 Stone 级）。

加载过程：遍历各 traits/ 目录 + views/ 目录，每个含 TRAIT.md / VIEW.md 的目录 = 一个 TraitDefinition。子目录递归处理（name 自动含 `/` 分级）。
