# 方法注册 — MethodRegistry 与沙箱调用

> Trait 暴露给 LLM 的不仅有知识（TRAIT.md），还有**方法**（可调用的函数）。

## 方法的两种来源

一个 trait 可以通过两种方式提供方法：

### 1. TRAIT.md 中声明的指令

通过 frontmatter 的 `command_binding`，让 trait 响应某些指令（如 program、talk、return）。当 LLM `open(command=X)` 并 submit 时，trait 的对应 handler 被调用。

### 2. methods.ts 中注册的函数

在 trait 目录下的 `methods.ts` 文件中，可以定义任意 TypeScript 函数。这些函数在 trait 激活时被注入**沙箱**：

```typescript
// kernel/traits/kanban/methods.ts
export const methods = {
  createIssue: {
    description: "创建一个新 Issue",
    params: [
      { name: "title", type: "string" },
      { name: "description", type: "string" },
    ],
    handler: async (ctx, { title, description }) => {
      // 实现
    },
  },
  // ...
};
```

LLM 在沙箱中可以直接调用：

```typescript
// LLM 生成的 program 代码
const issue = await createIssue({ title: "...", description: "..." });
```

## MethodRegistry

所有 trait 的 methods 在 trait 加载时注册到全局 **MethodRegistry**：

```typescript
class MethodRegistry {
  register(traitName: string, methods: Record<string, MethodDef>): void;
  getForTrait(traitName: string): Record<string, MethodDef>;
  // ...
}
```

## 沙箱方法的注入

每次构建沙箱时，**根据当前线程的 activatedTraits** 决定注入哪些方法：

```typescript
function buildSandboxMethods(activatedTraits: string[]): SandboxAPI {
  const methods = {};
  for (const traitName of activatedTraits) {
    const traitMethods = methodRegistry.getForTrait(traitName);
    Object.assign(methods, traitMethods);
  }
  return methods;
}
```

**关键**：未激活的 trait 的方法不会注入沙箱。这保证了：
- LLM 只能调用当前场景允许的方法
- 沙箱代码的可用 API 与 Context 中的知识一致

## 方法与指令的区别

| 维度 | 指令（command） | 方法（method） |
|---|---|---|
| 触发 | 通过 `open/submit` tool call | 在 program 代码中调用 |
| 粒度 | 粗粒度（program、talk、return） | 细粒度（createIssue、setData 等） |
| 层级 | LLM 直接交互 | 在 program 执行过程中调用 |
| 激活 | `open(type=command)` | 依赖 trait 激活状态 |

一个典型流程：

```
LLM: open(type=command, command=program)
      ↓ （激活 kernel/computable 及其方法）
LLM: submit(args: { code: "..." })
      ↓ （执行 program，沙箱中可调用 computable 方法）
sandbox: await writeFile(...)  # trait 方法调用
sandbox: await createIssue(...)  # 另一 trait 的方法（如激活了 kanban）
```

## 安全约束

### 1. 沙箱隔离

所有方法调用都在沙箱中执行（`kernel/src/executable/`）。沙箱：
- 不能访问任意文件系统
- 不能访问网络（除非特定 trait 明确开放）
- 不能 `require` 任意 npm 模块

### 2. 方法本身的权限

即使方法在沙箱中，它也可能操作持久化数据。每个方法的 handler 接收 `ctx`，含当前线程的身份信息，可用于权限检查：

```typescript
handler: async (ctx, args) => {
  if (ctx.currentObject !== "supervisor") {
    throw new Error("只有 Supervisor 可以创建 Issue");
  }
  // ...
}
```

### 3. trait 作用域

方法只有在 trait **当前激活**时可用。这意味着：
- LLM 不能在未激活 kanban trait 时调用 createIssue
- 即使历史上激活过，关闭后也不能再调

## 跨对象方法调用

通过 `talkable/cross_object` 子 trait，可以调用其他对象的 public 方法：

```typescript
// 调用对象 B 的 someMethod
const result = await talk_sync("B", { method: "someMethod", args: {...} });
```

这本质上是一次 talk_sync，对方 Flow 收到后解析 method + args，调用对应方法，返回结果。

详见 [../../合作/消息/跨对象协作.md](../../合作/消息/跨对象协作.md)。
