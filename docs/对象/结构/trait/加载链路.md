# Trait 加载链路

> Trait 以三层覆盖的方式加载：kernel → library → stone。后者覆盖同名前者。

## 三层加载顺序

```
1. kernel/traits/          ← 内置（基座 + 能力）
            ↓
2. library/traits/         ← 公共库（跨对象共享）
            ↓
3. stones/{name}/traits/   ← 对象自定义
```

加载顺序**从低优先级到高优先级**。后者覆盖前者——这是 OOC 的"扩展点"机制。

## 为什么这样分层

### kernel/traits/ — 系统内核

定义"作为 OOC 对象意味着什么"。所有对象共享这一层。

**修改意味着修改 OOC 本身**。除非做基因变更，否则不动。

例：`kernel/base`、`kernel/computable`、`kernel/talkable`...

### library/traits/ — 公共能力

可跨对象复用的能力。像 npm package：一次实现，多处使用。

**新增不影响现有对象**。对象只有主动在 readme 中激活某个 library trait 才会用上。

例：`library/browser`（网页浏览能力）、`library/lark-doc`（飞书文档读写）...

### stones/{name}/traits/ — 对象自定义

某个对象特有的能力。只属于这一个对象。

**增加不影响其他对象**。这是对象"成长"的物理载体（G12：trait 的沉淀就写在这里）。

例：Supervisor 专属的 `stones/supervisor/traits/session-kanban`。

## 覆盖规则

如果同名 trait 出现在多层：

```
kernel/traits/computable/           ← TRAIT.md 版本 A
library/traits/computable/          ← TRAIT.md 版本 B（覆盖 A）
stones/alan/traits/computable/      ← TRAIT.md 版本 C（覆盖 B）
```

Alan 对象加载到的是版本 C；Bruce 对象（没有自己的 computable）加载到版本 B；World（只加载 kernel）加载到版本 A。

**覆盖是整体的**，不是字段级的 merge——新版本完全替换旧版本。

### 何时会用覆盖

- 某个对象想**个性化某个内置能力**（罕见，谨慎使用）
- 某个对象想**屏蔽某个公共 trait**（通过本地提供同名空实现）

## 加载实现

`kernel/src/trait/loader.ts` 的入口：

```typescript
function loadAllTraits(): TraitRegistry {
  const registry = new Map();

  // 1. kernel/traits/
  loadFromDir("kernel/traits/", registry);

  // 2. library/traits/
  loadFromDir("library/traits/", registry);

  // 3. 各 stone 的 traits/（按需，仅加载当前 Session 用到的对象）
  for (const stoneName of activeStones) {
    loadFromDir(`stones/${stoneName}/traits/`, registry);
  }

  return registry;
}
```

每个 `loadFromDir` 遍历目录，识别含 TRAIT.md 的子目录，建立 TraitDefinition。
同名 trait 后加载的直接 `registry.set(name, def)` 覆盖。

## 加载时机

- **启动时** — 加载 `kernel/traits/` + `library/traits/`
- **对象首次被用到** — 加载该对象的 `stones/{name}/traits/`
- **热重载**（开发时）— TRAIT.md 变更触发 reload

## 实际验证

```bash
# 查看当前存在的 kernel traits
ls kernel/traits/

# 查看 library traits（如有）
ls library/traits/ 2>/dev/null

# 查看某对象的自定义 traits
ls stones/supervisor/traits/
```

## 源码锚点

| 概念 | 实现 |
|---|---|
| 加载入口 | `kernel/src/trait/loader.ts` |
| 路径解析 | `kernel/src/trait/loader.ts` → `resolvePath()` |
| 覆盖逻辑 | 默认 Map.set 行为（后覆盖前） |
| 热重载 | 目前通过进程重启触发 |

## 与基因的关联

- **G3**（trait 是自我定义）— 三层覆盖让"自我定义"既有默认又可个性化
- **G12**（经验沉淀）— 对象的成长写在 `stones/{name}/traits/` 里，是可覆盖层的物理体现
