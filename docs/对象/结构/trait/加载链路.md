# Trait 加载链路

> Trait 以四层覆盖方式加载：kernel → library → stone(trait) → stone(view)。后者覆盖同 traitId 前者。
> 2026-04-21 起，views 与 traits 共用同一张表（view 是 kind=view 的 trait）。

## 四层加载顺序

```
1. kernel/traits/                  ← 内置（基座 + 能力）          → namespace=kernel
            ↓
2. library/traits/                 ← 公共库（跨对象共享）         → namespace=library
            ↓
3. stones/{name}/traits/           ← 对象自定义 trait             → namespace=self, kind=trait
            ↓
4. stones/{name}/views/            ← 对象自渲染 view（UI + ui_methods）→ namespace=self, kind=view
            ↓
5. flows/{sid}/objects/{name}/views/  ← Flow 级 view（Session 内覆盖 Stone 级）
```

加载顺序**从低优先级到高优先级**。同 traitId（`namespace:name`）后者覆盖前者。

**加载入口**：`loadAllTraits(objectDir, kernelDir, libraryDir?, flowObjectDir?)`（kernel/src/trait/loader.ts）。

## 为什么这样分层

### kernel/traits/ — 系统内核

定义"作为 OOC 对象意味着什么"。所有对象共享这一层。

**修改意味着修改 OOC 本身**。除非做基因变更，否则不动。

例：`kernel/base`、`kernel/computable`、`kernel/talkable`...

### library/traits/ — 公共能力

可跨对象复用的能力。像 npm package：一次实现，多处使用。

**新增不影响现有对象**。对象只有主动在 readme 中激活某个 library trait 才会用上。

例：`library/browser`（网页浏览能力）、`library/lark-doc`（飞书文档读写）...

### stones/{name}/traits/ — 对象自定义

某个对象特有的能力。只属于这一个对象。

**增加不影响其他对象**。这是对象"成长"的物理载体（G12：trait 的沉淀就写在这里）。

例：Supervisor 专属的 `stones/supervisor/traits/session-kanban`。

## 覆盖规则

**覆盖基于 traitId（namespace:name）**。同一 traitId 不同位置时后加载者整体替换先加载者。

注意：新协议下 namespace 由 frontmatter 显式声明，所以实际极少发生真正的覆盖——
`kernel/traits/computable/` 声明 namespace=kernel → traitId=`kernel:computable`，
而 `stones/alan/traits/computable/` 只能声明 namespace=self → traitId=`self:computable`，
两者 traitId 不同，不会互相覆盖。

真正会覆盖的场景：Flow 级 view 覆盖 Stone 级同名 view（traitId 都是 `self:{viewName}`）。

**覆盖是整体的**，不是字段级的 merge——新版本完全替换旧版本。

### 何时会用覆盖

- 某个对象想**个性化某个内置能力**（罕见，谨慎使用）
- 某个对象想**屏蔽某个公共 trait**（通过本地提供同名空实现）

## 加载实现

`kernel/src/trait/loader.ts` 的入口（2026-04-21 新签名）：

```typescript
async function loadAllTraits(
  objectDir: string,          // 对象根目录（其下 traits/ + views/ 会分别扫）
  kernelTraitsDir: string,    // kernel/traits/
  libraryTraitsDir?: string,  // library/traits/（可选）
  flowObjectDir?: string,     // flows/{sid}/objects/{name}/（可选，用于 flow 级 views）
): Promise<{ traits: TraitDefinition[]; tree: TraitTree[] }>;
```

流程：
1. `loadTraitsFromDir(kernelTraitsDir, "kernel")` → 装入 traitMap
2. `loadTraitsFromDir(libraryTraitsDir, "library")` → 装入 traitMap（同 traitId 覆盖）
3. `loadTraitsFromDir(objectDir/traits, "self")` → 装入 traitMap
4. `loadObjectViews(objectDir)` → stone 级 views（kind=view）装入
5. `loadObjectViews(flowObjectDir)` → flow 级 views（可选，覆盖 stone 级）

每个 `loadTraitsFromDir` 递归扫目录，按文件名识别（TRAIT.md / VIEW.md / SKILL.md / readme.md）。
`loadObjectViews` 专扫 `{dir}/views/*/VIEW.md`，强制校验同目录存在 frontend.tsx。

## 加载时机

- **启动时** — 加载 `kernel/traits/` + `library/traits/`
- **对象首次被用到** — 加载该对象的 `stones/{name}/traits/`
- **热重载**（开发时）— TRAIT.md 变更触发 reload

## 实际验证

## 与基因的关联

- **G3**（trait 是自我定义）— 三层覆盖让"自我定义"既有默认又可个性化
- **G12**（经验沉淀）— 对象的成长写在 `stones/{name}/traits/` 里，是可覆盖层的物理体现
