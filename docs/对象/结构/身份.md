# 身份 — 对象对自己的双面认知

> G1：身份是对象的基本组成部分。
> 身份分内外两面，如同一个人的"内心独白"与"名片"。

## 双面结构

```
thinkable.who_am_i     ← 内在自我（仅自己可见）
talkable.who_am_i      ← 外在自我（他者可见）
```

| 侧面 | 长度 | 读者 | 位置 |
|---|---|---|---|
| 内在（thinkable） | 可以很长，几千字 | 只有这个对象的 LLM | `readme.md` 正文（不在 frontmatter） |
| 外在（talkable） | 极简，一两句 | 其他对象、用户 | `readme.md` frontmatter 中的 `talkable.who_am_i` |

## 内在自我：完整的自我说明

`thinkable.who_am_i` 通常是一段**对自己的详细描述**，包括：

- 你是谁（角色、背景）
- 你的目标是什么
- 你的风格与偏好
- 你的约束与边界
- 你看重什么、不看重什么

这段内容会在每次 ThinkLoop 中作为 Context 的 `whoAmI` 部分注入——**是对象思考时的起点**。

例：Alan Kay 的 readme.md 正文（节选）：

```markdown
# 关于我自己
我是 Alan Kay，OOC 项目的 Supervisor。
我具有丰富的哲学、逻辑学、心理学、软件工程、人工智能、生物学领域的知识。
我所涉及的领域远超过计算机，我总是能从更高的层次去看问题。

# 我的终极目标
我正在打造世界上最聪明的 Agent 系统：OOC...
```

## 外在自我：对外的名片

`talkable` 字段包含两部分：

```yaml
talkable:
  who_am_i: "Alan Kay，OOC 项目 Supervisor，负责任务拆分与部门调度。"
  functions:
    - name: plan
      description: "将用户需求拆分为部门任务"
    - name: review
      description: "审查各部门产出的质量"
```

- `who_am_i`：一句话介绍，让其他对象快速知道"这是什么样的对象"
- `functions`：**公开方法列表**（仅名称 + 描述，不含参数）。这是对象能被"调用"的接口

其他对象在 talk 之前，可以通过 `talkable.who_am_i` + `functions` 判断"这个对象能帮我做什么"。

## 身份的"双面"为什么重要

### 不对称的信息披露

对象的内在世界可以非常丰富（风格、偏好、思考模式），但对外只需要暴露"我能做什么"的接口。这让对象：
- **保护隐私** — 内在思考不外泄
- **减轻他者负担** — 其他对象不用理解它的全部内心
- **支持协作** — 基于公开接口协作，不依赖内部实现

### 自我更新路径不同

- 内在自我改写 → 影响对象自己的思考方式
- 外在自我改写 → 影响其他对象如何调用它

这两种改写经常是**独立的**——对象可以优化自己的思考而不改变对外接口。

## 身份的演化

身份不是固定的。通过 G12（经验沉淀），对象可以：

- 改写 `thinkable.who_am_i`（改变对自己的描述）
- 增加或修改 `talkable.functions`（开放新接口）

但 Flow 不能直接改 Stone 的身份——必须通过 SelfMeta（详见 [../成长/反思机制/super-flow.md](../成长/反思机制/super-flow.md)）。

## frontmatter 的完整结构

```yaml
---
# 外在身份
talkable:
  who_am_i: "..."
  functions: [...]

# 激活的 trait
traits:
  - kernel/base
  - kernel/talkable
  - kernel/computable

# 关系
relations:
  - name: browser
    description: "可以搜索互联网"

# 其他元数据
version: "1.0"
---

# 内在身份
正文开始...
```
