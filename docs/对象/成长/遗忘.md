# 遗忘 — G5 的另一面

> G5：Context 是对象每次思考时看到的全部信息。
> 推论：**Context 之外的一切，对象不知道**——不是"知道但不用"，而是**真的不知道**。

## 遗忘不是缺陷

传统软件的"遗忘"是 bug：数据丢了、缓存失效、对象被 GC。要修复。

OOC 的"遗忘"是**设计**。G5 说：**"遗忘是智能的基础设施"**。

### 为什么

LLM 的 Context 有硬上限（几万 token）。如果对象的所有历史都塞进 Context：
- 容量立刻爆
- 大部分信息与当前任务无关
- LLM 的注意力被无关信息稀释

**让不相关的信息退场**，才能让当前任务被看清。

## 三种遗忘

### 1. 时间维度：三层记忆

```
recent  ← 最近 N 轮完整
mid     ← 中段压缩为摘要
far     ← 远段极简归档或丢弃
```

越老的 actions，越被压缩。详见 [../认知/context/三层记忆.md](../认知/context/三层记忆.md)。

这是**自动遗忘**——不需要对象主动做什么。

### 2. 空间维度：作用域

```
子线程看不到兄弟的 actions
子线程看不到父的 process 细节
线程 A 看不到线程 B 的 inbox
```

这是**结构性遗忘**——不同线程之间物理上隔离。详见 [../认知/context/scope-chain.md](../认知/context/scope-chain.md)。

### 3. 意义维度：主动 mark ignore

```
inbox 消息用 mark(ignore) 标记为无需关注
```

这是**主动遗忘**——对象自己判断"这条消息不重要"。详见 [../认知/指令系统/mark.md](../认知/指令系统/mark.md)。

## 遗忘 vs 沉淀

这两个概念**互补而非矛盾**：

```
沉淀 = "我记住这件事的规律"
遗忘 = "我不再记得这件事的细节"
```

对象可以**沉淀规律，遗忘细节**：

```
经历：调试了一次内存泄漏，用 heap snapshot 找到了
  ↓
沉淀：memory.md 增加一条："heap snapshot 适合找内存泄漏"
  ↓
遗忘：具体哪次、哪个文件、哪个函数——Context 中不再有
```

这正是**压缩**——保留信息的**价值**（规律），丢弃信息的**量**（细节）。

## 什么不该被遗忘

### readme.md 的 whoAmI

身份应该**始终在 Context**。对象不应该"忘记自己是谁"。
（但 readme 可以被 SelfMeta 改写——这是成长，不是遗忘。）

### 激活的 trait

当前激活的 trait 应该**始终在 Context**（Level 1 或 Level 3）。对象不应该"忘记自己能做什么"。

### 当前线程的 actions（recent 区）

最近几轮的思考和行动应该**完整保留**。对象不应该"忘记自己刚才在做什么"。

## 什么应该被遗忘

### 远古任务的细节

上次 Session 的每一步 tool call——不需要记住。只需要 Flow Summary（一句话）。

### 已处理的 inbox 消息

mark ack 后，下一轮 Context 里还在，但 LLM 知道"这条已处理"，不再重复处理。

### 临时的中间状态

完成任务后，program 中用的中间变量、临时文件——不需要沉淀。

## 遗忘的可控性

对象可以在一定范围内**控制自己的遗忘**：

| 操作 | 效果 |
|---|---|
| `updateSessionMemory(...)` | 主动写 Session memory（避免被遗忘） |
| `talk(target="super", content="请记住 X")` | 请求沉淀到 Self（跨 Session 不忘） |
| `mark(ignore)` | 主动标记消息为可遗忘 |
| `updateFlowSummary(...)` | 写 Session 的一句话摘要（保留关键点） |

但**不能**控制的：
- 三层记忆的自动压缩节奏（由 Context 构建器统一）
- 线程 scope chain 的隔离（由架构决定）

## 遗忘的哲学意涵

### 遗忘让对象**可以成长**

如果对象记住所有经历的所有细节：
- Context 被旧任务淹没
- 新任务被旧经验的偏见影响
- 对象"越来越老"，失去面对新情况的灵活性

**遗忘细节，保留规律**——让对象既有经验又能适应。

### 遗忘让对象**保持对齐**

如果对象记得所有细节：
- 与人类的心理模型不符（人类也是遗忘细节的）
- 人类看不懂对象为什么"在意"一些陈年旧事

**恰当的遗忘让对象更像"有经验的人"**，而不是"永不遗忘的机器"。
