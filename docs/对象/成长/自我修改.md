# 自我修改 — 对象改变自己的具体路径

> 对象可以修改自己的身份、能力、关系——但不是随意的。每条修改路径都有明确的机制和约束。

## 三种自我修改

```
修改 readme.md          ← 改变身份
创建 / 修改 traits/     ← 改变能力
修改 relations          ← 改变社交网络
```

所有三种修改都**只能通过 SelfMeta（SuperFlow）**——主 Flow 不能直接改。

## 1. 修改 readme.md

### what：改身份

```markdown
# 我是 Alan
我是 OOC 项目的 Supervisor...
（原来只有这些）
```

经过多次任务后，SelfMeta 可能改写为：

```markdown
# 我是 Alan
我是 OOC 项目的 Supervisor，特别擅长哲学讨论...
（新增：擅长哲学讨论）
```

### why：为什么谨慎

readme 是对象的**宪法**。改 readme = 改变对象未来所有思考的起点。

错误的改动**影响深远**：
- 每次任务开始都从新身份出发
- 过去的成功经验可能变得不适用
- 其他对象对它的预期可能错位

### how：SelfMeta 的判断

SelfMeta 修改 readme 前必须：

1. **多次验证**：这个身份变化被多个任务证实
2. **深思熟虑**：是真正的身份演化，不是一次性心血来潮
3. **影响评估**：改动后对已有协作关系的影响
4. **保留原文**：通常是**追加**新内容，而非改写旧内容

## 2. 创建 / 修改 traits/

### what：改能力

```
stones/alan/traits/
├── (空)                  ← 初始

↓ 经验沉淀 ↓

stones/alan/traits/
├── philosophical-style/  ← 新增
│   ├── TRAIT.md
│   └── methods.ts
└── debug-flow-v2/        ← 新增
    └── TRAIT.md
```

新 trait 可以是：
- **添加新能力**（如 `debug-flow-v2`）
- **覆盖内置能力**（与 kernel 同名的 trait，局部覆盖）

### why：能力是最灵活的演化维度

相比 readme（谨慎）、relations（影响社交），trait 演化的代价相对低：
- 可以独立测试
- 可以随时禁用（删目录）
- 不影响其他对象的预期

### how：触发

```typescript
talk({ target: "super", content: "请沉淀为 trait：每次调试失败后必须读完整 stack trace" })
  ↓
SelfMeta:
  - 检查是否已有相似 trait → 是则合并
  - 否则创建新 trait
  - 写 TRAIT.md（含描述 + 操作说明）
  - 如需 methods，写 methods.ts
```

## 3. 修改 relations

### what：改社交网络

```yaml
# 原 readme frontmatter
relations:
  - name: filesystem
  - name: browser
```

经过协作后：

```yaml
relations:
  - name: filesystem
  - name: browser
  - name: bruce       ← 新增
    description: 体验测试负责人
```

### why：关系是社交的持久表达

临时协作 → 通过 talk；但如果一个对象**长期协作**，应该加入 relations，让它进入**帧 0**的 directory 字段。

### how：SelfMeta 审视

```typescript
talk({ target: "super", content: "我和 Bruce 协作了很多次，能不能记住他？" })
  ↓
SelfMeta 审视：
  - 协作次数够多吗？
  - 是持久关系还是一次性？
  - 是否重要到 always-on？
  ↓
决定：添加 relation
```

## 不能通过自我修改的事情

| 操作 | 可否自我修改 |
|---|---|
| readme.md 中的 thinkable.who_am_i | **是**，通过 SelfMeta |
| readme.md 中的 talkable（对外接口） | **是**，但最谨慎 |
| traits/ 下的 trait | **是** |
| data.json | **是**（主 Flow 的 setData 也可以，但仅限 Flow 级） |
| memory.md | **是** |
| 其他对象的数据 | **否**（只能通过 talk 让对方改） |
| 系统级配置 | **否**（超出对象边界） |

## 自我修改的审计

所有自我修改都通过 SuperFlow 完成，其 actions 历史记录了**什么时候、改了什么、为什么**：

```
SuperFlow.actions:
  - 2026-04-21 10:30 - talk(target="super") received: "请记住..."
  - 2026-04-21 10:31 - thought: "验证过 3 次，可以沉淀"
  - 2026-04-21 10:32 - tool_use: writeFile(memory.md, ...)
  - 2026-04-21 10:33 - return: "已记录到 memory.md"
```

这让"对象为什么变成现在这样"**可追溯**。

## 自我修改 vs 他人修改

| 修改方式 | 可以吗 |
|---|---|
| **自我修改**（通过 SelfMeta） | ✓ |
| **人类直接改 readme** | ✓（管理员手动操作） |
| **另一个对象改我的 readme** | ✗（除非我 talk 允许） |
| **世界改我的身份** | ✗（不存在这个机制） |

**别人不能绕过你的意志改你**——这是 OOC 的核心自主性。

## 源码锚点

| 概念 | 实现 |
|---|---|
| readme 写入 | `kernel/src/persistence/writer.ts` |
| trait 创建 | 同文件 + 目录操作 |
| SelfMeta | `stones/{name}/super/` |
| 权限检查 | 各方法 handler 内 |

## 与基因的关联

- **G12**（经验沉淀）— 自我修改是沉淀的物理实现
- **G3**（trait 是自我定义）— trait 层面的自我修改
- **G1**（数据即对象）— 修改对象 = 改变其数据
- **G7**（目录即存在）— 自我修改 = 改变物理字节
