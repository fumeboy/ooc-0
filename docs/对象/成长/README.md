# 成长 — 对象如何"变"

> G12：对象通过经历改写自身结构。
> **这是 OOC 最激进的哲学主张**：认知结构不是先天固定的，而是在经验中成长的。

## 最激进的主张

传统 AI 系统的"对象"是固定的——它们的身份、能力、记忆由程序员写死。如果想改变对象，需要程序员修改代码、重新部署。

**OOC 坚持对象可以改写自己**。不是通过代码修改，而是通过**它自己的经历**。这个过程是 G12（经验沉淀）。

## 沉淀循环

G12 的核心是**沉淀循环**：

```
经历 → 记录（G10） → 反思（talk(super)） → SuperFlow 审视 → 沉淀为 trait → 改变帧 0
  ↑                                                                          │
  └──────────────────── 新的经历在新的起点上发生 ─────────────────────────────┘
```

每转一圈：
- 对象的 `readme` 或 `traits` 可能被改写
- 下一次任务开始时，对象"是不同的自己了"
- 这个"不同"来自真实的经验，不是外部赋予

## 三层结构

沉淀不是单一的——分为三个层次：

```
知识（记住发生了什么） → memory.md / actions 摘要
能力（知道怎么做）     → trait 的 methods + readme
直觉（不用想就做）     → always-on trait（升级后）
```

每一层都是**更高阶的内化**。见 [三层结构/](三层结构/)。

## 反思的角色

**反思不是自动的**。不是每个经历都会改变对象——只有经过反思、被 SuperFlow 审视的经验才会沉淀。

这个设计保证了：
- 对象不会因为一次错误就全盘重写自己
- 对象不会把幻觉沉淀为"真理"（verifiable trait 把门）
- 沉淀是深思熟虑的结果，不是数据量的积累

详见 [反思机制/](反思机制/)。

## 遗忘是成长的前提

G5 说"遗忘是智能的基础设施"——不是丢失，是让不相关退场。

成长必须伴随遗忘。如果对象记住所有经历，它的 Context 永远被旧任务的细节塞满，无法专注当前任务。

详见 [遗忘.md](遗忘.md)。

## 与其他维度的关系

- **存在**（readme + data）：成长改写的是**存在的物理字节**
- **结构**（trait）：成长的重要载体
- **认知**（context）：沉淀后的"新自己"在新一轮 Context 中体现
- **合作**（消息）：经验往往来自合作（他人的反馈 / 纠正 / 启发）
