# 沉淀循环 — G12 的动力学

> 经验沉淀不是一次性事件。它是一个**循环**——每次转一圈，对象厚一点。

## 完整循环

```
  ┌──────────────────────────────────────┐
  │                                      │
  ↓                                      │
经历  ──→  记录  ──→  反思  ──→  沉淀  ──→  新的起点
(G10)    (actions) (super)   (readme/trait)
```

每一圈的产物是**对象自身结构的改变**——新的起点不再是旧的起点。

## 五个阶段

### 1. 经历

主 Flow 在执行任务中发生各种事情：
- 调用了某个 API，返回了某种结果
- 与其他对象沟通，收到某种反馈
- 犯了一个错误，从错误中学到了什么
- 做对了一件事，发现了某个规律

这些是**原始经历**——尚未被任何意识编码。

### 2. 记录

G10：所有经历被**不可变地记录**为 actions：

```json
{ "type": "tool_use", "tool": "...", "args": {...}, "result": {...}, "ts": "..." }
{ "type": "thought", "content": "...", "ts": "..." }
{ "type": "text", "content": "...", "ts": "..." }
```

写入 `thread.json.actions`。这是事实层——发生了就是发生了。

### 3. 反思

主 Flow 在某个时刻（通常是 return 前）意识到："这次有些值得记住的"：

```
talk(target="super", content="请记住：debugger 比 print 快，我应该优先用 debugger")
```

消息投递到 SuperFlow。

### 4. 沉淀

SuperFlow 审视：
- **真实性**：有证据吗？有验证吗？
- **价值**：值得永久记住吗？
- **形式**：记到哪里？（memory / readme / trait）

判断后：
- 不值得 → 拒绝，或只记入 Session memory（临时）
- 值得 → 写入相应位置

**产物**：
- `stones/{name}/memory.md` 多了一段话
- 或 `stones/{name}/readme.md` 的正文被改写
- 或 `stones/{name}/traits/xxx/` 被创建/修改

### 5. 新的起点

下一次主 Flow 被唤醒（同 Session 下一轮，或未来某次 Session），初始 Context：
- whoAmI（来自新的 readme）
- knowledge（来自新激活的 trait）
- 长期记忆（来自新的 memory）

**对象不是同一个对象了**——虽然 name 相同、目录相同，但里面的字节已不同。

## 循环的两种粒度

### 微循环（任务内）

一次 Session 里转多圈：
- 遇到错误 → talk(target="super") → SuperFlow 更新 memory
- 下一个子任务时已经记得错误

这是**短时记忆**的成长。

### 宏循环（跨 Session）

多次 Session 积累：
- 本次发现的规律，下次 Session 开始时已在 Context
- 逐渐从 memory → trait → 默认激活 trait（直觉）

这是**长期成长**。

## 沉淀循环 vs 展开循环

[../../哲学/两个循环.md](../../哲学/两个循环.md) 讲到 OOC 的两个循环：

- **展开循环**：身份 → 能力 → 计划 → 思考 → 行动
- **沉淀循环**：行动 ← 记录 ← 反思 ← 沉淀 ← 身份

```
身份 → 能力 → 计划 → 思考 → 行动
  ↑                              │
  └── 沉淀 ← 反思 ← 记录 ←───────┘
```

展开让对象**做事**；沉淀让对象**变成新的自己**。两个循环一起转，对象才是"活的"。

## 反循环的风险：错误沉淀

```
错误的经历 → 错误的反思 → 错误的沉淀 → 下次 Context 里是错误的身份
```

如果对象沉淀了错误的"经验"（如基于幻觉的 trait），它会**每一次思考都被误导**——对象越来越笨。

防御（在 G12 + verifiable 之上）：

- **verifiable**：沉淀前要有验证证据
- **SuperFlow 的 Bias**：保守、诚实、克制
- **可撤销**：对象可以通过 talk(target="super") 请求移除一个 trait，或"重写"memory（罕见）

## 沉淀的速度

**不应该每次任务都沉淀**。

健康的节奏：
- 大部分任务 = 纯粹做事，不沉淀
- 偶尔 = 反思，但仅写 Session memory
- 更偶尔 = 沉淀到 Self（memory 或新 trait）
- 很少 = 升级 trait 为默认激活（直觉）

如果每次都沉淀，memory 和 traits 会膨胀，反而降低对象的效率。
