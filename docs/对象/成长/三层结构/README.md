# 三层结构 — 知识 → 能力 → 直觉

> 经验沉淀分三个层次。每一层是对上一层的内化。

## 三层对应

| 层次 | 含义 | 载体 | 激活方式 |
|---|---|---|---|
| [知识](知识.md) | 记得发生了什么 | memory.md + actions 历史 | 被动：LLM 回顾时看到 |
| [能力](能力.md) | 知道怎么做 | trait（有 methods） | 按需：open 时激活 |
| [直觉](直觉.md) | 不用想就做 | 默认激活 trait | 自动：每轮都在 Context |

## 递进关系

```
知识     "上次遇到 X 时，Y 方法有效"
  ↓ （多次重复验证）
能力     "遇到 X 时，用 Y 方法" → 写成 trait 的方法
  ↓ （长期使用，证明稳定）
直觉     trait 升级为默认激活 → 不用想都在 Context 里
```

## 为什么分三层

### 知识是**碎片**

一次经验 = 一条知识。"上次这么做对了"、"某人说了某话"。知识不构成"能力"——它只是素材。

### 能力是**模式**

从多次知识中提炼出"遇到 X 就该做 Y"的模式。写成 trait 的 readme + methods，成为可复用的能力。

### 直觉是**默认**

能力被反复使用后，证明稳定且重要。它应该升级为"不需要每次思考时主动激活"——变成默认激活，进入每一轮 Context 的基础背景。

## 沉淀的方向：越来越主动

每一层的激活方式不同：

- **知识** — 对象自己读 memory.md 才看到
- **能力** — 对象 `open(title="...", command=X, description="...")` 触发 trait 激活
- **直觉** — 每轮 Context 都有，不需要主动

**直觉 = 把重复主动变成系统自动**。这是 G12 沉淀循环的终点。

## 三层与"帧 0"

G12 讨论中常提到"帧 0"——对象启动新任务时的初始 Context。

- 每次帧 0 都有的东西 → **直觉**（默认激活 trait）
- 可能在帧 0 中被引用到的东西 → **能力**（readme 中 traits 列表）
- 仅在某次任务中回忆起的东西 → **知识**（memory.md）

好的成长 = **把最重要的东西升级到直觉**，让对象的"起跑线"越来越高。

## 反例：错误的沉淀

不是所有经验都该沉淀为更高层次：

| 经验 | 该沉淀到？ |
|---|---|
| 一次成功的代码实现 | 知识（"这次我写出了这个函数"） |
| "遇到 API 错误要读完整错误信息" | 能力（trait：debuggable）|
| "用 print 而不是 console.log" | 直觉（写入默认激活的核心 trait） |
| 一次错误的判断 | **不应该沉淀** — 知识层面记录即可 |
| 一次未经验证的结论 | **不应该沉淀** — verifiable 把门 |

错误的沉淀会**污染**未来的直觉。SuperFlow 的核心职责就是**选择什么值得沉淀**。

## 跨三层的工具

OOC 提供几个工具跨三层工作：

| 工具 | 作用层 |
|---|---|
| `updateSessionMemory(text)` | 知识（Session 内） |
| `talk(target="super", content="请记住 X")` | 知识（Self 级，跨任务） |
| `talk(target="super", content="请沉淀为 trait：X")` | 能力 |
| 手动编辑 readme 的 traits 列表 | 能力 → 直觉（改 trait 的 when） |
