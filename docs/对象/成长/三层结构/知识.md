# 知识 — 记得发生了什么

> 沉淀的第一层。最原始、最直接。

## 含义

**知识** = 具体事件的记录。"上次遇到 X 时，Y 方法有效"。

不是抽象的规律，是**具体的事实**。

## 载体

| 位置 | 时效 |
|---|---|
| **actions 历史** | 线程级，任务内可见 |
| **Session memory**（`flows/{sid}/objects/{name}/memory.md`）| Session 内 |
| **Self memory**（`stones/{name}/memory.md`）| 永久（跨所有任务） |

### actions 历史

每一轮 ThinkLoop 的 thought + tool_use + 结果都作为 action 追加到 `thread.json`。

这是**最粒度细**的知识——对象当时想了什么、做了什么、得到什么。但体量大，不适合长期保留。

### Session memory

当前 Session 的精华：

```markdown
# 会话记忆

## 2026-04-21 10:30
在线程树集成时，发现 issue-discussion 实际位置与 meta.md 不符。
验证方法：ls kernel/traits/。

## 2026-04-21 11:15
用户澄清了"工程管理"目录的设计意图是"元设计"。
```

由对象自己写（通过 `updateSessionMemory`）。Session 结束即消散（不写入 Self）。

### Self memory

Stone 级的**长期记忆**。只能通过 reflect() 写入：

```
reflect("请记住：issue-discussion 是 talkable 的子 trait")
  → SelfMeta 审视
  → 可能写入 stones/{name}/memory.md
```

写入后**跨所有任务可见**——下次 Alan 的任何 Session 都能看到这段记忆。

## 知识 vs 能力

```
知识：上次遇到 A，我用 B 解决了
能力：遇到 A，用 B（trait: when_see_A, do_B）
```

知识是**事件**；能力是**模式**。

### 什么时候该沉淀为能力

当某个知识**重复出现**、**规律稳定**、**适用面广**时——它应该从知识升级为能力。

例子：
- "这次调试我用了 print 定位问题" → 知识
- "这次又用了 print" → 还是知识，但开始形成模式
- "第 5 次了，所有调试任务都该先 print" → **升级为能力**（写入 debuggable trait）

决策权在 ReflectFlow：它看到"这个知识是否值得变成能力"。

## 知识的注入路径

每一轮 Context 构建时：

```
process（actions 历史）← 来自当前 thread.json
Session memory  ← 作为独立字段或补充 whoAmI
Self memory     ← 作为 whoAmI 的一部分
```

知识**默认**进入 Context。但 Context 容量有限，所以：

- actions 走三层压缩（见 [../../认知/context/三层记忆.md](../../认知/context/三层记忆.md)）
- Session memory 通常保留最近 N 次更新
- Self memory 全部保留（数量级可控，因为写入门槛高）

## 知识的"真实性"保障

G10（行动记录不可变）保证 actions 历史真实。
verifiable trait 保证沉淀到 Self memory 的知识经过验证。
Comment 不可变保证 Issue 下的讨论真实。

**诚实的知识是下一层（能力、直觉）的地基**——地基不真，上层都歪。
