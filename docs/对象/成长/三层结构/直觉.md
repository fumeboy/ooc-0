# 直觉 — 不用想就做

> 沉淀的最高层。能力被内化为"默认方式"。

## 含义

**直觉** = 不需要主动激活就生效的能力。

对象甚至**意识不到**它在遵循这个规则——因为规则已经是 Context 的一部分，就像呼吸一样自然。

## 物理形式

**对象默认 trait**：

```yaml
---
name: alan-custom/communication-style
type: how_to_think
---
```

对象默认 trait 意味着：只要对象把它列入默认激活清单，每一轮 Context 都注入它的 readme。

## 基础直觉：kernel/base

最基础的直觉是 `kernel/base`（指令系统基座）。每个 OOC 对象都以 base 为直觉。

base 定义了对象"如何与系统交互"——open、submit、close、wait。对象在思考时**不需要思考**"我该怎么调用工具"——它就是知道。

## 对象特定的直觉

对象可以有自己的默认 trait：

```
stones/alan/
├── readme.md  (traits 列表中包含 alan-custom/communication-style)
└── traits/
    └── alan-custom/
        └── communication-style/
            ├── TRAIT.md
            └── ...
```

alan 的 "communication-style" 始终影响他的每一轮思考——如何回复消息、如何整理信息、用什么语气。这成为 alan 的**默认行为模式**。

## 升级路径：能力 → 直觉

当一个能力被反复验证稳定且重要，它可以升级为**直觉**：把它加入对象默认激活清单，并确认 readme 的内容适合始终注入。

升级**谁来决定**？

- 对象自己通过 reflect 请求升级
- Sophia（哲学设计层）评估是否该升级
- 人类手动修改

升级**有代价**：
- Context 多一份内容（占 token）
- 影响所有任务（不仅是相关任务）

所以**不是越多直觉越好**。直觉清单应该精简，每条都值得始终存在。

## 直觉与"帧 0"

帧 0 = 对象开始新任务时的初始 Context。直觉是帧 0 的核心组成。

```
对象 Alan 的帧 0：
  - whoAmI: Alan 的 readme 正文
  - instructions: base + alan-custom/communication-style
  - （尚未激活任何能力层 trait）
```

帧 0 的厚度 ≈ 对象的"经验厚度"。

**新手对象**：帧 0 很薄，只有 base。做事需要开很多子线程、激活很多 trait。
**专家对象**：帧 0 厚，各种直觉已经内化。同样的任务用浅的线程树就能完成。

这是 G12 沉淀循环在**运行时可观测**的最终体现——**经验厚 → 树浅**。

## 直觉的陷阱

### 过度沉淀

如果什么都沉淀为直觉，Context 会被填满，反而稀释了对关键信息的注意力。

**经验法则**：直觉 ≤ 5 条对象特定 trait。超过就该反思"哪些该回到能力层"。

### 沉淀错的直觉

如果一个基于幻觉的"能力"升级为直觉，它会**每一轮都在误导**对象。

防御：
- verifiable 把门（沉淀前要有验证）
- 升级前要多次验证能力的稳定性
- 定期检视直觉清单，淘汰不再有效的

## 直觉的可观察性

前端在 Stone 详情页会展示对象默认激活的 trait：

```
Traits（始终激活）:
  - kernel/base
  - kernel/talkable
  - alan-custom/communication-style
Traits（按需激活）:
  - kernel/computable
  - kernel/reflective
  - ...
```

这让人类能看到"Alan 的直觉清单"——他把什么内化为默认。

## 源码锚点

| 概念 | 实现 |
|---|---|
| 默认激活 vs 按需激活 | 对象默认激活清单 + `activates_on.show_*_when` |
| 始终注入 | `kernel/src/thread/context-builder.ts` |
| readme 的 traits 列表 | `kernel/src/persistence/frontmatter.ts` |

## 与基因的关联

- **G3**（trait 是自我定义）— 直觉是 trait 的最高形态
- **G12**（经验沉淀）— 沉淀的终点
- **G5**（Context 即世界）— 直觉是帧 0 的核心组成
