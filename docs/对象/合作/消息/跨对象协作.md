# 跨对象协作 — Session + 线程树联动

> 当一个任务涉及多个对象时，它们如何在同一个 Session 内协同工作。

## 基本图景

```
Session sess_xyz
  ├── Flow (supervisor)    线程树 A
  ├── Flow (alan)           线程树 B
  ├── Flow (filesystem)     线程树 C
  └── Flow (bruce)          线程树 D

  Flow 之间通过 talk / talk_sync 联络
  线程树 A.root 发送消息到 B.root → B 启动处理
```

## 协作如何开始

### 从 Supervisor 分发

```
用户消息 → supervisor.root.inbox
supervisor 处理：
  - 分析需求
  - 决定需要哪些对象参与
  - talk(alan, "请设计 X") → alan.root 启动
  - talk(bruce, "之后你来验证") → bruce.root 启动
```

### 从任意对象发起

任何对象都可以主动 talk 其他对象：

```
alan 在执行过程中发现需要 filesystem 帮忙：
alan.thread: open(talk_sync) + submit({ target: "filesystem", ... })
  ↓
filesystem.Flow 自动创建（如不存在）
filesystem.root 收到消息，处理，return
  ↓
alan.thread 的 talk_sync 收到回复
```

## Flow 自动创建

协作的核心便利：**talk 触发自动创建 Flow**。

```typescript
// collaboration.ts 的逻辑
async function deliverMessage(sessionId, target, message, from) {
  // 检查目标 Flow 是否存在
  let flow = await getFlow(sessionId, target);
  if (!flow) {
    // 不存在 → 自动创建
    flow = await createFlow(sessionId, target);
  }

  // 写入目标 Flow 的根线程 inbox（或指定 thread）
  await writeInbox(flow, threadId, message, from);
}
```

这让协作"零配置"——只要知道对方的名字（且对方的 Stone 存在），就能联络。

## 线程作为话题通道

A 和 B 之间的**同一话题**应该在**同一棵线程**里进行：

```
首次：A.thread_1 talk(B, "讨论 X")
  → B.thread_X 启动（作为 A.thread_1 的"对话伙伴"）

后续：A.thread_1 再 talk(B, "关于 X 的新发现")
  → 消息路由到 B.thread_X（不是 B.thread_root，也不是 new thread）
  → 如果 B.thread_X 已 done，复活
```

这让"对话上下文"跨时间保持——B 不会忘记之前的讨论。

路由规则实现：`collaboration.ts` 维护 "A.thread → B.thread" 的映射。

## 三种协作模式

### 1. 并行独立

多个对象做互不相关的事：

```
supervisor 分发：
  talk(alan, "任务 A")
  talk(bruce, "任务 B")
  talk(charlie, "任务 C")
supervisor wait([alan_ret, bruce_ret, charlie_ret])
```

三个 Flow 并发，互不影响。

### 2. 串行协作

后一个依赖前一个：

```
supervisor:
  const design = await talk_sync(alan, "设计 X")
  const code = await talk_sync(coder, { design })
  const review = await talk_sync(reviewer, { code })
```

用 talk_sync 链式依赖。

### 3. 讨论式

多对象在一个 Issue 下讨论：

```
supervisor 创建 Issue (话题)
  - alan 评论（意见 A）
  - bruce 评论（意见 B）
  - supervisor 综合 → 推进到下一步
```

看板（Issue/Task）提供了结构化的多方讨论。详见 [../结构化/](../结构化/)。

## Session 的边界

**一个 Session = 一次任务 = 一组协作的边界**。

- 同 Session 内的对象互相知道（通过 Session objects 目录）
- 跨 Session 的对象需要重新 talk 启动新 Flow

**Session 结束**：Supervisor 判定任务完成，或用户终止。所有 Flow 清理。

## 协作中的 trait 共享

同一 Session 内，多个对象的 Flow 都可以独立激活自己的 trait。**不共享**激活状态：

```
alan.thread 激活了 library/browser
bruce.thread 没激活 browser
  → bruce 不能用 browser API
```

这是合理的——trait 是对象能力，不是 Session 能力。
