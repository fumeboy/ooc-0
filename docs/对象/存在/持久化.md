# 持久化 — 目录即存在

> G7：**对象的持久化目录就是它的物理存在**。

## 一个对象 = 一个目录

在 OOC 中，不存在"内存里的对象"这种东西。任何时候你说到"对象 X"，它指的都是**文件系统上的一个目录**。

```
stones/
├── alan/
│   ├── .stone                    ← 标记文件：这个目录是一个 Stone
│   ├── readme.md                 ← 身份
│   ├── data.json                 ← 动态数据
│   ├── memory.md                 ← 长期记忆
│   ├── traits/                   ← 能力定义
│   ├── reflect/                  ← ReflectFlow 的持久化目录
│   ├── ui/
│   │   └── index.tsx             ← 自渲染 UI（可选）
│   └── files/                    ← 其他共享文件
├── bruce/
├── supervisor/
└── ...
```

- `alan/` 目录存在 → 对象 `alan` 存在
- `alan/` 被删除 → 对象 `alan` 消亡
- `mv alan/ charlie/` → 对象被重命名（需同步更新 readme.md）

## 关键文件

| 文件 | 作用 | 必需？ |
|---|---|---|
| `.stone` | 标记此目录是一个 Stone 对象 | **必需** |
| `readme.md` | 身份定义（`who_am_i`、traits 激活列表） | **必需** |
| `data.json` | 动态键值对数据 | 可选 |
| `memory.md` | 长期记忆（跨任务存在） | 可选 |
| `traits/` | 用户自定义 Trait 目录 | 可选 |
| `reflect/` | ReflectFlow 子对象目录 | 可选 |
| `ui/index.tsx` | 自渲染 UI 入口 | 可选 |
| `files/` | 共享文件 | 可选 |

`.stone` 文件是身份标记。遍历 `stones/` 时，系统用它来识别哪些子目录是对象、哪些是普通目录。

## 为什么是文件系统

OOC 选择文件系统作为对象的物理载体，不是因为文件系统最好用，而是因为它是**唯一满足四个要求的载体**：

1. **可读** — 人类能直接打开、查看、备份
2. **可写** — 对象可以修改自己的身份、数据、能力
3. **可引用** — 路径即地址，可以被其他对象（或 LLM）引用
4. **可归档** — 目录能复制、删除、迁移

数据库不满足 1（不可读）；内存变量不满足 4（无持久化）；vector store 不满足 3（无稳定地址）。

## 推论 1：对象可以"搬家"

```bash
cp -r stones/researcher/ /backup/researcher-backup/
mv stones/researcher/ teams/alpha/researcher/   # 重新挂载即可
```

对象是**可迁移的实体**。团队协作时可以直接 commit Stone 目录到 git。

## 推论 2：对象的"死亡"是物理的

删除目录不只是"逻辑删除"——文件系统里真的不存在了。对应的 memory.md、traits/、ui/、reflect/ 全部消失。这与 `哲学/emergences/death_legacy.md` 中"对象的遗产"讨论一致。

## 推论 3：对象的"身份改变"是原地的

修改 `readme.md` 中的 `who_am_i`，对象的身份**立刻**改变——下次思考时，LLM 看到的就是新身份。不需要"迁移数据"、"更新索引"、"通知依赖方"。文件改了就是改了。

## Flow 的持久化

Flow 不在 `stones/` 下，而在 `flows/{sessionId}/objects/{name}/`：

```
flows/
└── sess_20260421_abc/             ← 一个 Session = 一个目录
    ├── .session.json              ← Session 元数据
    ├── readme.md                  ← Session 工作状态摘要（supervisor 维护）
    ├── objects/
    │   ├── alan/                  ← Alan 在这个 Session 的 Flow
    │   │   ├── .flow              ← 标记：Flow 存活
    │   │   ├── data.json          ← Flow 的运行时数据
    │   │   ├── threads.json       ← 线程树索引
    │   │   ├── threads/
    │   │   │   └── {threadId}/
    │   │   │       └── thread.json  ← 单线程的 actions、locals、plan
    │   │   ├── memory.md          ← 会话记忆
    │   │   └── ui/pages/          ← Flow 演示页面
    │   ├── bruce/
    │   └── ...
    ├── issues/                    ← 看板：Issue 跟踪
    │   ├── index.json
    │   └── issue-{id}.json
    └── tasks/                     ← 看板：Task 跟踪
        ├── index.json
        └── task-{id}.json
```

详见 [session.md](session.md)。

## 源码锚点

- `kernel/src/persistence/reader.ts` — 读取 Stone / Flow / Thread 数据
- `kernel/src/persistence/writer.ts` — 写入（含原子写 + 并发串行化）
- `kernel/src/persistence/frontmatter.ts` — readme.md 的 frontmatter 解析
- `kernel/src/persistence/thread-adapter.ts` — 线程树 → 旧 Process 格式转换（过渡）
