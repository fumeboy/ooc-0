# 子线程

> 通过 `create_sub_thread` 在线程树下创建新节点。子线程独立执行，完成后 return 结果给父线程。

## 为什么需要子线程

### 保持焦点

当前线程在做 A 任务，中途发现需要做 B 子任务。如果在同一线程做 B，B 的思考会污染 A 的 Context。

**解法**：开子线程做 B。B 完成后只把摘要返回给 A。A 不看 B 的具体过程。

### 作用域隔离

子任务可能需要**不同的 trait 作用域**。比如 A 线程在写代码，需要临时做一次代码审查——审查要激活 reviewable，但审查完就该卸载。

**解法**：开子线程做审查。reviewable 在子线程激活，子线程 return 后自动卸载。

### 并行

多个独立子任务可以并行开子线程。父线程 `wait_all` 等全部完成。

## 创建子线程

```
open(type=command, command=create_sub_thread)
submit(form_id, {
  title: "搜索相关文档",
  description: "在 docs/ 下找与 X 相关的 md 文件，返回前 5 个最相关",
  inherit_scope: true  // 默认 true
})
→ 返回 thread_id: "th_xxx"
```

## 参数

| 参数 | 含义 | 必填 |
|---|---|---|
| `title` | 子线程标题（显示用） | 是 |
| `description` | 任务描述（作为 parentExpectation 传给子线程） | 是 |
| `inherit_scope` | 是否继承父的 trait 激活 | 可选，默认 true |
| `initial_data` | 初始数据（注入子线程 locals） | 可选 |

## 生命周期

```
父线程 create_sub_thread
    ↓
 创建子节点 (status=running)
    ↓
 Scheduler 开始调度子线程
    ↓
 子线程独立执行（新的 ThinkLoop 循环）
    ↓
 子线程 return
    ↓
 子线程 status=done
 return summary 写入父线程 inbox
    ↓
 父线程收到 inbox 消息（如果在 wait，则恢复 running）
```

## inherit_scope

### inherit_scope: true（默认）

子线程**继承父线程的 trait 激活状态**：

```
父: activatedTraits = {base, talkable, computable}
子（inherit=true）: activatedTraits = {base, talkable, computable}（scope chain 继承）
```

子线程可以直接用这些能力，不需要重新 open。

### inherit_scope: false

子线程是**独立作用域**：

```
父: activatedTraits = {base, talkable, computable}
子（inherit=false）: activatedTraits = {base}  (only)
```

子线程需要自己 `open` 需要的 command / trait。

**典型场景**：创建一个"独立身份"的子任务，不想被父线程的 domain knowledge 影响。

## 与 talk 的关系

`create_sub_thread` 和 `talk` 的区别：

| 维度 | create_sub_thread | talk |
|---|---|---|
| 目标 | 同一对象的子任务 | 另一个对象 |
| 作用域 | 继承父（默认） | 目标对象的独立作用域 |
| 通信 | 父子直接（通过 inbox） | 跨对象通信 |
| 场景 | 任务拆分 | 能力调用 / 协作 |

什么时候用哪个？

- 任务在**当前对象的能力范围内** → 开子线程
- 任务需要**另一个对象的专业能力** → talk

## 等待子线程

```
// 等待单个
wait(["th_xxx"])

// 等待多个（全部完成）
wait(["th_xxx", "th_yyy"])

// 等待任意一个
// 用 Promise.race 或自己管理
```

`wait` 让父线程进入 waiting。子线程完成后自动唤醒。

## continue_sub_thread

向已创建的子线程**追加消息**：

```
open(type=command, command=continue_sub_thread)
submit(form_id, {
  thread_id: "th_xxx",
  message: "继续：再检查 Y"
})
```

如果子线程已 done，自动**复活**（详见 [线程复活.md](线程复活.md)）。

## 子线程的 Context

子线程第一轮思考时的 Context：

- **whoAmI**：同对象 readme（同父）
- **parentExpectation**：子线程的 description（来自 create_sub_thread 的参数）
- **process**：空（刚开始）
- **inbox**：空
- **activeForms**：空
- **directory**：同对象目录
- **instructions + knowledge**：按 inherit_scope 决定
- **childrenSummary**：空

子线程的 Context **不包含**父线程的 process / inbox / activeForms。这保证了独立性。
