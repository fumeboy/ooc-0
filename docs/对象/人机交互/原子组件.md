# 原子组件 — 最小可复用 UI 单元

> 跨页面复用的基础组件。这里**不是给每个组件单独写文档**——它们的接口从源码可见，这里汇总说明各组件的**目的和设计取舍**。

## ObjectAvatar — 对象头像

**确定性颜色 + 首字母圆形头像**。

- 基于 name 的 hash 选择颜色（同名永远同色）
- 首字母作为文字
- 大小可配置（16px / 24px / 40px）
- **无需后端数据**——纯函数

```tsx
<ObjectAvatar name="alan" size={40} />
```

## Badge — 语义化彩色标签

- **StatusBadge** — Flow 状态（running / waiting / finished / failed / pausing）
- **ActionBadge** — Action 类型（thought / program / message_in 等）

用色彩传达语义：
- 绿 = 正常
- 橙 = 处理中
- 灰 = 静止
- 红 = 异常

## MarkdownContent — Markdown 渲染器

统一的 Markdown 展示。特性：

- GFM（GitHub Flavored Markdown）
- 代码高亮（highlight.js）
- 表格
- **ooc:// 链接拦截**——点击触发 OocLinkPreview

## CodeMirrorViewer — 代码查看器

只读代码展示（CodeMirror 6）。支持：

- JSON（JSON 高亮 + 折叠）
- JS / TS（语法高亮）
- Markdown（双模式：渲染 vs 源码）
- 纯文本

## CodeBlock — 简单代码块

轻量级 `<pre>`，支持限高（超过后加 scroll）。比 CodeMirror 轻，适合对话中引用代码。

## Sheet — 侧滑面板

从屏幕边缘滑出的面板。用法：

- ActionCard Zoom-in
- OocLinkPreview
- Session Detail 侧栏
- ...

支持：
- 左 / 右 / 底部滑出
- 自定义高度 / 宽度
- 背景遮罩（点击关闭）

## FloatingGradient — 浮动渐变背景

三色光球装饰。用于 WelcomePage、空白页面的背景——让空白不显得死板。

## ErrorBoundary — React 错误捕获

捕获子组件抛错，展示友好的错误页：

```
出了点问题
[具体错误]
[Retry 按钮]
```

应用在：
- DynamicUI（自渲染 UI 失败）
- 各主要页面外围

## 为什么这些组件独立存在

它们都是**跨页面重复使用**的。抽成独立组件：

- DRY（避免重复实现）
- 统一设计语言（所有 avatar 一个样式）
- 单点维护（改一处影响全局）
