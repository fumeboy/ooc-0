# 自渲染 — UI 即对象的面孔（G11）

> 对象不是"被 UI 设计"的，它**自己画自己**。

## 路径

### Stone 级：唯一入口

```
stones/{name}/ui/index.tsx
```

每个 Stone 最多一个 `ui/index.tsx`——它是对象的"主界面"。

### Flow 级：多页面

```
flows/{sid}/objects/{name}/ui/pages/
  ├── task-001-report.tsx
  ├── issue-002-summary.tsx
  └── progress.tsx
```

Flow 通常用于**报告**：展示某个 Task 的结果、某次实验的可视化等。不是主界面，是"加页"。

## 组件契约

### Stone index.tsx

```tsx
// stones/alan/ui/index.tsx
import React from "react";

export default function AlanView({ stone, session }) {
  return (
    <div>
      <h1>Alan's Custom UI</h1>
      <StoneReadme stone={stone} />
      {session ? <SessionView session={session} /> : null}
    </div>
  );
}
```

Props：
- `stone` — 当前对象的完整 StoneData（readme、data、relations 等）
- `session` — 当前活跃 Session（可能为 null，表示没有活跃 session）

### Flow page.tsx

```tsx
export default function TaskReportView({ flow, task, session }) {
  return (
    <div>
      <h2>任务结果：{task.title}</h2>
      <Chart data={task.result} />
    </div>
  );
}
```

Props 包含 Flow 数据和关联的 Task / Issue。

## 动态加载

前端用 Vite 的 `import` 动态加载：

```typescript
// kernel/web/src/components/DynamicUI.tsx
async function loadUIComponent(path: string) {
  try {
    const module = await import(/* @vite-ignore */ path);
    return module.default;
  } catch (error) {
    console.error("UI load failed", error);
    return null;  // 降级到 fallback
  }
}
```

`@vite-ignore` 告诉 Vite 不做静态分析（因为路径是运行时决定的）。

## 热更新

开发时，修改 `ui/index.tsx` → Vite HMR 自动推送更新 → 前端立即看到新 UI。

对象修改自己的 UI（通过 program 写 `ui/index.tsx`）→ 同样触发 HMR。

## 失败降级

如果：
- 文件不存在
- 文件有语法错误
- 组件渲染抛错

→ **自动降级到通用视图**（ObjectDetail / SessionKanban 等）。用户不会看到"白屏 / 报错页"——只是看到默认的布局。

这是 `kernel/web/src/components/ErrorBoundary.tsx` 的职责。

## 有自定义 UI 的对象

前端检测到对象有 `ui/index.tsx`：
- **默认展示 UI Tab**（而不是 Readme Tab）
- **导航标题带特殊图标**（表示"这个对象有自定义界面"）

让用户知道这个对象不是"通用对象"——它有专属界面。

## 典型用例

### Supervisor: Session Kanban

Supervisor 的 `ui/index.tsx`：

```tsx
export default function SupervisorDashboard({ session }) {
  if (!session) return <WelcomePage />;
  return <SessionKanban sessionId={session.id} />;
}
```

用户打开 Supervisor 时，直接看到当前 Session 的看板。

### 数据可视化对象

一个"市场数据分析"对象的 `ui/index.tsx`：

```tsx
export default function MarketDataView({ stone }) {
  const data = JSON.parse(stone.data.latestMarket);
  return <LineChart data={data} />;
}
```

对象的 data.json 包含最新的市场数据，UI 直接展示为图表。**对象改 data → UI 自动更新**。

### 工作流面板

```tsx
export default function WorkflowPanel({ stone }) {
  const steps = stone.data.steps;
  return (
    <StepList>
      {steps.map(step => (
        <StepItem key={step.id} step={step} onDone={handleDone} />
      ))}
    </StepList>
  );
}
```

`onDone` 通过 API 触发某个 talk 到对象，对象收到消息后更新 data。

## 组件安全性

自渲染 UI 在浏览器中运行——有沙箱风险？

**当前实现**：自渲染 UI 和主 App 运行在**同一个 JS 上下文**中。这意味着：

- 自渲染 UI 可以访问 window、localStorage、fetch 等
- 可以调用主 App 的其他组件（通过 import）
- **没有强沙箱**

这是一个妥协：强沙箱（iframe / Web Worker）会极大限制灵活性。当前假设"对象的 ui/ 代码是可信的"（来自 Session，来自知道情况的用户或对象）。

未来可能加入可选的沙箱模式（如通过 iframe + postMessage）。
