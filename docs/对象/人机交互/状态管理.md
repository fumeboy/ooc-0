# 状态管理 — Jotai Atoms

> OOC 前端用 Jotai 做全局响应式状态。每个 atom = 一个独立可订阅的状态单元。

## 为什么用 Jotai

对比其他方案：

- **Redux**：概念重，boilerplate 多
- **Context API**：粒度粗，任意一处变更全树重渲染
- **Zustand**：简单，但没有 atom 粒度
- **Jotai** ✓：原子化、类型安全、依赖追踪自动

## 核心 atom 分组

### session atoms — Session/Flow/UI 核心状态

| Atom | 内容 |
|---|---|
| `activeTabAtom` | 当前 ModeSwitch 选中的 Tab（Flows / Stones / World） |
| `activeSessionIdAtom` | 当前活跃 Session ID |
| `activeSessionFlowAtom` | 当前活跃 Session 的 Flow 数据 |
| `editorTabsAtom` | 打开的文件标签列表 |
| `activeFilePathAtom` | 当前选中的文件路径 |
| `lastFlowEventAtom` | 最新 SSE 事件（驱动全局刷新） |
| `streamingTalkAtom` | 流式对话内容（实时展示） |
| `streamingThoughtAtom` | 流式思考内容 |
| `refreshKeyAtom` | 手动刷新计数器（触发重新拉取） |
| `messageSidebarOpenAtom` | MessageDock 展开/折叠状态 |

### objects atoms — Stone 对象列表

| Atom | 内容 |
|---|---|
| `objectsAtom` | 所有 Stone 的列表 |
| `selectedObjectAtom` | 当前选中的对象 |

### ooc-link atoms — ooc:// 链接弹窗状态

| Atom | 内容 |
|---|---|
| `oocLinkPreviewAtom` | 当前预览的链接（null = 关闭） |

## atom 的组合

Jotai 的核心优势：**derived atom** 自动追踪依赖。

```typescript
// 衍生 atom：当前 Session 的 Issues
const currentIssuesAtom = atom((get) => {
  const sessionId = get(activeSessionIdAtom);
  if (!sessionId) return [];
  return fetchIssues(sessionId);
});
```

`activeSessionIdAtom` 变化 → `currentIssuesAtom` 自动重新计算 → 订阅它的组件重渲染。

## atom 的持久化（可选）

部分 atom 通过 `localStorage` 持久化：

- `editorTabsAtom` — 关闭浏览器后恢复打开的 tab
- `activeTabAtom` — 记住用户上次在哪个 Tab

## 与 SSE 的配合

SSE 事件进入 → 通过 hook（如 `useSSE`）写入 atom：

```typescript
useSSE("flow:action", (event) => {
  setAtom(lastFlowEventAtom, event);
  // 其他订阅 lastFlowEventAtom 的 atom / 组件自动响应
});
```

详见 [sse.md](sse.md)。
