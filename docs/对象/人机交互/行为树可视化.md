# 行为树可视化（ProcessView）

> 对象思考过程的可视化。双栏布局：左 ActionTimeline + 右 MiniTree。

## 结构

```
┌──────────────────────────────┬─────────────────┐
│                              │                 │
│   ActionTimeline（左）       │   MiniTree      │
│                              │   （右）        │
│   ▸ thread-001               │                 │
│     ┌──────────┐              │   • thread-001  │
│     │ action 1  │             │     • thread-002│
│     └──────────┘              │     • thread-003│
│     ┌──────────┐              │                 │
│     │ action 2  │             │                 │
│     └──────────┘              │                 │
│     ...                      │                 │
│                              │                 │
└──────────────────────────────┴─────────────────┘
```

## ActionTimeline（左栏）

沿当前选中节点的 **scope chain 向上收集 actions**，按节点分组展示：

### NodeHeader

每个节点的头部：
- 状态圆点（绿=done, 橙=doing, 灰=pending）
- 标题
- focus 节点标记 "(focus-on)"
- action 计数

### ActionCard

每个 action 展示为一张卡片。详见 [卡片/README.md](卡片/README.md)。

### NodeSummary

如果节点的 actions 被压缩（三层记忆的中段/远段），展示为 NodeSummary 虚线边框卡片。

## MiniTree（右栏）

节点树缩略视图：

- 可展开/折叠
- 点击切换选中节点（左栏跟着变）
- 状态圆点：绿=done, 橙=doing, 灰=pending, 红=failed
- focus 节点标记 "(focus-on)"

## 交互

### 选中节点

默认选中根线程。点击 MiniTree 的任意节点 → 切换选中：
- ActionTimeline 显示该节点的 scope chain actions
- Breadcrumb 更新为节点路径

### Zoom-in action

ActionCard 的 "Zoom-in" 按钮 → 打开 Sheet 侧滑面板，显示 action 完整详情（包括未截断的参数、完整结果）。

### Copy / Ref

每个 action 有 Copy（复制内容）和 Ref（复制 ooc:// 链接到此 action）按钮。

## 为什么不是"完整行为树视图"

OOC 早期有"行为树视图"——把所有节点画成一张大图。问题：

- 节点多时难以浏览
- actions 在"图里的某个节点"里，双重层级
- 缩放/平移操作繁琐

**双栏视图**更实用：
- 左侧是"当前焦点的详细时间线"
- 右侧是"整体树形小地图"
- 结合两者，用户既能看局部又能看整体
