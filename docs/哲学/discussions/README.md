# 哲学讨论

> 还未定论的设计问题、正在讨论中的方向、有争议的观点。
> 这里的内容是 **append-only**：讨论一旦进入，就保留其历史原貌，即使结论后来改变。

## 为什么需要这个目录

哲学层的文档（`genes/`、`emergences/`、`统一性.md`、`两个循环.md`）是**已沉淀的共识**。
但 OOC 是一个活的系统，它的设计每天都在演化——有些变化是增量的（走代码修改），有些变化是**结构性的**（会撼动基因）。

结构性变化需要一个地方被**充分讨论**，而不是直接改基因。这个目录就是这样的地方。

## 写作约定

### 文件命名

`{日期}-{主题关键词}.md`，如 `2026-04-05-多对象上下文管理.md`。

### 文件结构

```markdown
# {主题}

> 讨论日期：{日期}  | 状态：{进行中 / 已结论 / 已搁置}
> 相关基因：{G3, G12, ...}
> 发起人：{Sophia / Alan Kay / ...}

## 问题陈述

（一段话说清要讨论什么）

## 关键观点

### 观点 A — {论点}
{论证}

### 观点 B — {论点}
{论证}

## 已达成的共识
（如有）

## 未决问题
（如有）

## 后续动作
（如有）
```

### Append-only 规则

- **新讨论** → 新建文件
- **讨论结论** → 在原文件加 "结论" 小节；不删改原有观点
- **基因变更** → 在 `../genes/` 下修改后，在这里留一条"已落入基因 Gx"的标注
- **搁置** → 状态改为"已搁置"，文件保留

## 与实验的关系

实验（`工程管理/验证/实验/`）验证的是**涌现能力是否真的出现**；讨论这里处理的是**基因设计本身是否合理**。

一个典型的流程是：

```
实验观察到异常 → 进入 discussions/ 讨论 → 形成共识 → 改基因 或 改实现
                                    └→ 未达成共识 → 保留讨论 → 待未来证据
```
