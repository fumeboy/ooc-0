# 13 条基因

> OOC 系统的最小规则集合。所有上层能力都是这些基因的组合与涌现，而非额外添加的特性。

## 基因总览

| 编号 | 名称 | 核心命题 |
|---|---|---|
| [G1](g01-数据即对象.md) | 数据即对象 | OOC 中的一切实体都是对象。对象就是 Agent 本身 |
| [G2](g02-stone-与-flow.md) | Stone 与 Flow | 对象分为两种形态：静态（Stone）与动态（Flow） |
| [G3](g03-trait-自我定义.md) | Trait 是自我定义单元 | 对象通过 trait 定义自己的能力；trait 可组合、可进化、自约束 |
| [G4](g04-输出程序以行动.md) | 对象通过输出程序来行动 | 所有行动 = 对象生成一段程序 + 沙箱执行 |
| [G5](g05-context-即世界.md) | Context 即世界 | 对象每次思考时看到的全部信息。对象不知道 Context 之外的任何事 |
| [G6](g06-关系即网络.md) | 关系即网络 | 对象通过关系连接成有向图；每个对象只知道自己的关系 |
| [G7](g07-目录即存在.md) | 目录即存在 | 对象的持久化目录就是它的物理存在；目录删除 → 对象消亡 |
| [G8](g08-effect-与-space.md) | Effect 与 Space | 对象作用于世界的唯一通道 |
| [G9](g09-行为树计划执行.md) | 线程树计划执行 | 对象的运行时 = 一棵线程树（旧称行为树）；一棵树 = 一次完整的任务处理 |
| [G10](g10-行动记录不可变.md) | 行动记录不可变 | 所有行动按时间顺序追加，历史从不改写 |
| [G11](g11-ui-即面孔.md) | UI 即面孔 | 对象的视觉呈现由对象自身数据直接生成 |
| [G12](g12-经验沉淀.md) | 经验沉淀 | 对象通过经历改写自身结构：知识 → 能力 → 直觉 |
| [G13](g13-认知栈即运行模型.md) | 线程树即运行模型 | 线程树是对象思考的统一运行模型（替代"认知栈 + 行为树"） |

## 命名规则

- 文件名格式：`g{NN}-{短名}.md`，其中 `NN` 是两位编号（便于排序）
- 短名用连字符分隔的中文关键词
- 每条基因独立成文，互相通过链接引用

## 阅读顺序建议

**按编号顺序**（推荐）：G1-G13 本身是一个逐步展开的叙述。G1 定义对象，G2-G3 说对象的形态与能力，G4-G5 说对象如何行动与思考，G6-G8 说对象之间的关系，G9-G10 说对象的执行机制，G11-G13 说对象的表达、成长与整体运行。

**按维度分组**（另一种读法）：
- **存在维度**：G1, G2, G7
- **结构维度**：G3, G6
- **认知维度**：G5, G13
- **行动维度**：G4, G8, G9, G10
- **成长维度**：G12
- **表达维度**：G11

## 与涌现的关系

基因是微观规则，涌现是宏观能力。每条基因单独看是简单的规定，但多条基因组合会产生基因本身不具备的能力——这些能力记录在 [../emergences/](../emergences/) 目录。

例如 E1（自我进化）由 G3（trait 可进化）+ G7（目录即存在）+ G12（经验沉淀）共同涌现。

## 重要说明

本目录下的基因内容从 2026-03 的 `gene.md` 拆分迁移而来。部分基因的描述（特别是 G9、G13）早于线程树架构重构，文字上仍保留"行为树"、"认知栈"等旧术语。**最新的实现语义以 `对象/` 下的工程文档为准**；基因文本作为**设计初心的历史快照**保留，后续重大变更应走 [../discussions/](../discussions/) 流程。
