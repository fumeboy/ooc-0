# E4: 记忆、经验与反思

**涉及基因**: G10(事件历史) + G3(Trait) + G4(程序行动) + G12(经验沉淀)

对象的行为树节点记录了完整的行为历史。当反思类 trait 被激活时：
1. 对象回顾当前节点的 actions
2. 发现模式（「我总是在第三步出错」）
3. 输出程序修改自己的 traits（「下次先检查前置条件」）

这是元认知的涌现：
- actions 提供了「经验数据」
- trait 提供了「反思框架」
- 程序行动提供了「自我修改能力」

三者结合，对象可以从经验中学习，而不仅仅是执行指令。

## 渐进沉淀路径（G12）

经验沉淀是 trait 在原地成长：
- 第一次完成挑战 → 创建 trait（只有 readme.md，记录经验）
- 第二次遇到类似 → 为 trait 添加 index.ts（封装能力）
- 第三次遇到类似 → 将 trait 的 when 改为 always（内化直觉）

每一层都是对上一层的验证。这避免了过早沉淀导致的错误。
