# E10: 对象的「死亡」与「遗产」

**涉及基因**: G7(持久化即存在) + G6(关系) + G10(事件历史)

当一个对象被销毁时：
- 它的持久化目录被删除（物理死亡）
- 但它的影响留在其他对象的 actions 中（记忆遗产）
- 它创建的对象仍然存在（创造遗产）
- 它沉淀的 traits 如果被其他对象引用，仍然有效（能力遗产）

一个对象可以「死去」，但它对系统的影响是永久的。
这是涌现的「历史性」——系统的当前状态是所有过去对象行为的累积结果。

## 新模型的变化

- Effects 目录归档 = Flow 的自然死亡（任务完成后资源回收）
- Stone 的死亡 = 整个 objects/{name}/ 目录删除
- Kernel traits 不会因为某个对象的死亡而消失（它们属于系统层）
