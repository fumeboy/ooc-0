# E6: 人机协作

**涉及基因**: G4(程序行动/可中断) + G10(事件历史) + G5(Context)

Flow 的 pausing 机制让人类可以介入对象的思考过程：
1. 人类暂停 Flow
2. 查看行为树了解对象的计划和执行进度
3. 注入 thought（提供新信息）或 program（代替执行）
4. 恢复 Flow，对象在新的 context 下继续思考

这不是「人类控制 AI」，而是「人类参与 AI 的思考过程」。
对象看到注入的事件后，会将其纳入自己的推理——
就像一个同事在你思考时提供了新的视角。

## 新模型的变化

行为树让人机协作更精确：
- 人类可以看到结构化的计划（而非扁平的 actions 列表）
- 人类可以直接修改行为树（添加/删除/调整节点）
- 人类可以移动 focus 光标，引导对象关注特定子任务
