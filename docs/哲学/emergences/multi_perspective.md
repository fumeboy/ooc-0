# E2: 多视角思考

**涉及基因**: G3(Trait 能力单元) + G5(Context 认知边界) + G9(行为树)

一个 Flow 面对复杂问题时：
1. 在行为树中创建多个子节点，每个子节点配置不同的 trait 激活组合
2. 每个子节点从不同角度分析同一个问题
3. 汇聚各子节点的完成摘要，形成综合判断

这不是简单的「多次调用 LLM」。
每个子节点有独立的 actions 和不同的 trait 激活配置，看到的 Context 是不同的。
同一个对象，通过行为树的不同分支，实现了「一人分饰多角」。

## 新模型的变化

旧模型通过 Thread 的 biasNames 切换视角。
新模型通过行为树节点的 activatedTraits 配置切换视角，更加灵活：
- 每个节点可以激活不同的 traits 组合
- 栈进/栈出自动管理视角切换
- 完成摘要自动汇聚到父节点
