# E1: 自我进化

**涉及基因**: G1(万物皆对象) + G4(程序行动) + G7(持久化即存在) + G12(经验沉淀)

对象可以输出程序来修改自己的 traits/ 目录，从而获得新的思维方式、知识或方法。
下一轮思考时，新能力已经可用。

这意味着：
- 一个 Flow 可以在运行时发现自己缺少某种能力，然后为自己创建新 trait
- 创建的 trait 被持久化到文件系统，重启后仍然存在
- 对象的能力边界不是静态的，而是随经验扩展的

极端情况：一个足够聪明的 Flow 可以重写自己的全部 traits，
本质上变成一个完全不同的对象——同一个 name，全新的行为。

## 新模型的变化

旧模型中自我进化分散在 codes/、biases/、windows/ 三个目录。
新模型中统一为 traits/，进化路径是 trait 的有机成长：
- 阶段 1：创建只有 readme.md 的 trait（知识）
- 阶段 2：为 trait 添加 index.ts（能力）
- 阶段 3：将 trait 的 when 改为 always（直觉）

trait 迭代时不保留旧版本，直接在原版本基础上修改。
