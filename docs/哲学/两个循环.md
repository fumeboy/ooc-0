# 两个循环 — OOC 的动力学

> 概念树不是静态的。它被两个循环驱动——**展开循环**让对象生长出行动，**沉淀循环**让经验反哺身份。

## 两个循环的图景

```
展开循环（正向）：
  身份(readme) → 能力(Trait) → 计划(线程树) → 思考(Engine) → 行动(Effect)
                                                                    │
沉淀循环（逆向）：                                                  │
  身份(readme) ← 沉淀(G12) ← 反思(reflect) ← 记录 ← 经验 ←─────────┘
```

对象从身份出发，展开为行动；行动的结果沉淀回身份。
这个循环每转一圈，对象的经验就厚一层。

## 展开循环

**起点**：对象被唤醒（用户消息 / talk / 子线程被创建）
**终点**：Effect 对外部世界产生影响

步骤：

1. **身份** — `readme.md` 中的 `who_am_i` 进入 Context，告诉对象"你是谁"
2. **能力** — 激活的 trait 决定对象"能做什么"（哪些指令可用、哪些知识可见）
3. **计划** — 线程树结构化地表达"怎么做"（子线程 / await / return）
4. **思考** — ThinkLoop 的单轮循环，LLM 基于 Context 生成下一个动作
5. **行动** — 输出的程序在沙箱执行，产生 Effect（文件、消息、新对象）

每一步都依赖上一步：没有身份就没有能力方向；没有能力就没有计划素材；没有计划就没有思考的焦点；没有思考就没有行动。

## 沉淀循环

**起点**：行动产生了结果（成功 / 失败 / 意外）
**终点**：身份或能力发生了改变

步骤：

1. **经验** — 行动的结果写入 Context 和 thread.json（不可变，G10）
2. **记录** — actions 历史成为经验的原始材料
3. **反思** — 对象调用 `reflect(message)`，触发 ReflectFlow
4. **沉淀** — ReflectFlow 审视经验，判断是否值得改写 readme / trait / memory
5. **身份** — 下一次展开循环时，对象已经是"新的自己"

关键特性：**沉淀不是自动的**。并不是所有经验都会改变对象。只有通过反思、经过"自我对话"的经验，才会真正沉淀下来（详见 G2 中关于 SelfMeta 的阐述）。

## 为什么两个循环都需要

单独看每个循环都不完整：

- **只有展开循环** → 对象永远是出厂设置，不会变聪明
- **只有沉淀循环** → 对象没有经历可沉淀，沉淀循环空转

**两个循环一起转，对象才变成一个"活的"存在**：它既能做事（展开），又能从做事中变成一个更好的自己（沉淀）。

## 与"两个循环"对应的工程设计

| 循环 | 对应工程模块 |
|---|---|
| 展开循环 · 身份 | `kernel/src/persistence/reader.ts` 读 readme.md |
| 展开循环 · 能力 | `kernel/src/thread/hooks.ts` 的 collectCommandTraits |
| 展开循环 · 计划 | `kernel/src/thread/tree.ts` + scheduler.ts |
| 展开循环 · 思考 | `kernel/src/thread/engine.ts` 的单轮循环 |
| 展开循环 · 行动 | `kernel/src/executable/` 沙箱执行 |
| 沉淀循环 · 经验 | `kernel/src/persistence/writer.ts` 写 actions |
| 沉淀循环 · 反思 | `kernel/traits/reflective/` |
| 沉淀循环 · 沉淀 | ReflectFlow 对 readme / traits / memory 的写入 |

## 两个循环与"成长"维度的关系

沉淀循环 = 成长维度的工程表达。详见 [../对象/成长/](../对象/成长/)。

展开循环则分布在四个领域：存在（readme） → 结构（trait） → 认知（线程树/thinkloop） → 合作（effect/消息）。

## 推论：设计检验法

每当设计一个新机制时，检验它是否符合 OOC 哲学的一个快速方法是问：

- 它参与展开循环吗？
- 它参与沉淀循环吗？
- 如果都不参与，它**不是 OOC 对象的一部分**，可能需要重新思考定位。

例如：日志系统本身不在任何一个循环中——但 actions 历史在（沉淀循环的"经验"环节）。所以 OOC 中不存在独立的"日志系统"，只有"actions 持久化"。
