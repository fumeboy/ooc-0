# E12: 对象的「面孔」—— UI 涌现

**涉及基因**: G11(UI 自我表达) + G4(程序行动) + G7(持久化即存在) + G3(Trait)

当对象获得 UI 能力后，涌现出一种新的存在形式：

1. **自我定义展示方式**
   - 对象通过 `ui_template` kernel trait 获得编写 UI 的能力（方法 + 指导）
   - 对象通过 program 输出创建 `ui/index.tsx`
   - UI 是对象的顶层组成部分（与 readme.md、data.json 平级），不是 trait 的附属品
   - 这是「我→我」Effect 的延伸：对象决定自己如何被看见

2. **从被动到主动**
   - 没有 UI 时：人类通过 API 查询对象状态
   - 有 UI 时：对象主动呈现最重要的信息
   - 类似于：一本书（被动）vs 一个讲解员（主动）

3. **UI 与能力的共进化**
   - 对象获得新 trait → 更新 UI 展示新功能
   - 对象改变身份 → 更新 UI 匹配新角色
   - UI 成为对象「自我认知」的外部投射

4. **前端成为 World 的一部分**
   - 前端项目位于 `.ooc/web/`，与对象共存于 World 目录
   - 前端可以动态加载任何对象的 UI
   - 这打破了「后端对象 vs 前端展示」的二元对立

涌现效果：
- 每个 UI 都是独特的——由对象自己编写，反映其个性和功能
- 人类不需要学习每个对象的 API——对象通过 UI 主动提供交互界面
- 系统整体的可视化是所有对象 UI 的涌现结果，而非一个中心化的设计
