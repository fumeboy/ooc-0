# E5: 计划与执行的分离

**涉及基因**: G9(行为树) + G3(Trait/planning) + G5(Context)

planning trait 引导 Flow 先制定计划，再逐步执行：
1. 分析任务 → 一次性创建行为树（可以很庞大）
2. 按 focus 光标逐步执行各节点
3. 每完成一个节点，自动栈出回到父节点
4. 遇到问题时，随时调整行为树结构
5. 所有计划和执行状态持久化在 process.json 中

涌现效果：
- 行为树作为结构化 plan 始终可见，对象不会「忘记计划」
- 每个节点有独立的 actions 和 trait 激活配置，上下文清晰
- 计划可以在执行过程中动态调整（不是瀑布式的）
- focus 光标的栈进/栈出自动实现结构化遗忘

## 新模型的变化

旧模型通过 Thread + plan.md（windows/）实现计划。
新模型用行为树（process.json）替代，优势：
- 计划和执行状态统一在一个结构中（不再分散在 Thread 和 windows/ 里）
- focus 光标提供精确的注意力控制
- deps 支持节点间的依赖关系
- 最大深度 20 层，防止过度分解
